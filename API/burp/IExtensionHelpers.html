インタフェース IExtensionHelpers


  • public interface IExtensionHelpers
    Buurp拡張で起こる様々な一般的なタスクの支援に便利な、数々のヘルパーメソッドが含まれています。拡張からIBurpExtenderCallbacks.getHelpersを呼び出すと、このインタフェースのインスタンスを取得できます。

      • stringToBytes

        byte[] stringToBytes(java.lang.String data)
        データを文字列からバイト配列に変換します。特定の文字セットは変換に反映されず、16進数で0xWXYZの文字は常に0xYZを表すバイトに変換されます。bytesToString()メソッドとは反対の変換を行い、この二つのメソッドを使い文字列に変換したバイトベースのデータとそれを戻したデータは、完全性の維持が保証されます(ただし指定された文字セットを反映する変換では当てはまりません)。
        パラメータ:
        data - 変換するデータ。
        戻り値:
        変換されたデータ。

      • bytesToString

        java.lang.String bytesToString(byte[] data)
        バイト配列から文字列形式にデータを変換します。特定の文字セットは変換に反映されず、16進数で0xYZの文字は常に0x00YZを表すバイトに変換されます。stringToBytes()メソッドとは反対の変換を行い、この二つのメソッドを使い文字列に変換したバイトベースのデータとそれを戻したデータは、完全性の維持が保証されます(ただし指定された文字セットを反映する変換では当てはまりません)。
        パラメータ:
        data - 変換するデータ。
        戻り値:
        変換されたデータ。

      • indexOf

        int indexOf(byte[] data,
            byte[] pattern,
            boolean caseSensitive,
            int from,
            int to)
        指定したパターンがデータ内で最初に出現する部分を検索します。文字列ベースのデータに対して動作するJavaのネイティブメソッド String.indexOf()と同じように、バイトベースのデータで動作します。
        パラメータ:
        data - 検索するデータ。
        pattern - 検索するパターン。
        caseSensitive - 検索で大文字小文字を区別するかどうかのフラグ。
        from - 検索を開始するdataのオフセット。
        to - 検索を終了するdataのオフセット。
        戻り値:
        指定された範囲で最初にパターンが出現したオフセット、または見つからない場合は-1。

      • buildHttpMessage

                        byte[] body)
        指定されたヘッダとメッセージ本文を含むHTTPメッセージを構築します。必要に応じて、ボディの長さに基づいてContent-Lengthヘッダが追加または更新されます。
        パラメータ:
        headers - メッセージに含めるヘッダのリスト。
        body - メッセージのボディ、ない場合はnull。
        戻り値:
        結果として得られる完全なHTTPメッセージ。

        パラメータ:
        url - リクエストを作成するURL。
        戻り値:
        指定したURLへのリクエスト。

      • addParameter

        byte[] addParameter(byte[] request,
                    IParameter parameter)
        HTTPリクエストに新しいパラメータを追加し、必要に応じてContent-Lengthヘッダを更新します。
        パラメータ:
        request - パラメータを追加するリクエスト。
        parameter - 追加するパラメータの詳細を含んだIParameterオブジェクト。サポートしているパラメータ形式は次の通りです: PARAM_URL、PARAM_BODY、PARAM_COOKIE。
        戻り値:
        新しいパラメータが追加された新規HTTPリクエスト。

      • removeParameter

        byte[] removeParameter(byte[] request,
                       IParameter parameter)
        HTTPリクエストからパラメータを削除し、必要に応じてContent-Lengthヘッダを更新します。
        パラメータ:
        request - パラメータを削除するリクエスト。
        parameter - 削除するパラメータの詳細を含んだIParameterオブジェクト。サポートしているパラメータ形式は次の通りです: PARAM_URL、PARAM_BODY、PARAM_COOKIE。
        戻り値:
        パラメータが削除された新規HTTPリクエスト。

      • updateParameter

        byte[] updateParameter(byte[] request,
                       IParameter parameter)
        HTTPリクエスト内のパラメータ値を更新し、必要に応じてContent-Lengthヘッダを更新します。注: このメソッドは、指定されたタイプのパラメータが存在する場合のみ使用できます。既存のパラメータのタイプを変更する必要がある場合は、まずremoveParameter()を呼び出して古いタイプのパラメータを削除し、その後addParameter()を呼び出して新しいタイプのパラメータを追加してください。
        パラメータ:
        request - 更新するパラメータを含んだリクエスト。
        parameter - 更新するパラメータの詳細を含んだIParameter オブジェクト。サポートしているパラメータ形式は次の通りです: PARAM_URL、PARAM_BODY、PARAM_COOKIE。
        戻り値:
        パラメータが更新された新規HTTPリクエスト。

      • toggleRequestMethod

        byte[] toggleRequestMethod(byte[] request)
        リクエストのメソッドのGETとPOSTを切り替えます。必要に応じてパラメータは、URLクエリーストリングとメッセージボディ間で再配置され、Content-Lengthヘッダは適切に作成または削除されます。
        パラメータ:
        request - メソッドを切り替えるHTTPリクエスト。
        戻り値:
        メソッドが切り替えられた新規HTTPリクエスト。
