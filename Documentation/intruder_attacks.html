攻撃実行

全ての攻撃設定が完了したら、攻撃の開始をし、結果を分析し、場合によって攻撃設定を修正し、テストワークフローとのリンクやその他のアクションを実行する必要があります。

攻撃の開始

2 つの方法で、攻撃を開始できます:

  • Target、ポジション、ペイロード、オプションの各タブで攻撃設定をし、Intruderメニューから"攻撃開始"を選択します。
  • Intruderメニューから"Open saved attack"を選択し、以前保存した攻撃を開きます。

それぞれの攻撃は別のウィンドウで開かれます。このウィンドウには生成された攻撃の結果が表示され、攻撃設定をリアルタイムに修正でき、テストワークフローとのリンクやその他のアクションの実行もできます。

結果タブ

結果タブには、攻撃で発行された全リクエストの完全な詳細があります。これらの情報にフィルターや注釈を付けると分析しやすくなり、テストワークフローの推進にも利用できます。

結果一覧

結果の表には、攻撃で生成された全リクエストとレスポンスの詳細が表示されます。表には次の列がありますが、攻撃設定によっていくつかはデフォルトで非表示になっていて、列メニューで表示させられます:

  • リクエスト番号。攻撃で元のリクエストをそのまま送るような設定がされていたら、番号０のアイテムとして一覧内に表示されます。
  • 現在のペイロードのポジション番号(スナイパー攻撃用)
  • リクエストで使われたペイロード
  • HTTP ステータス コード
  • リクエストが行われた時刻
  • レスポンスを受信し始めるまでの所要時間(ミリ秒)
  • レスポンスを受信し終わるまでの所要時間(ミリ秒)
  • リクエスト送信中にエラーが発生したかどうか
  • レスポンスや処理を待っている間タイムアウトが発生したかどうか
  • レスポンスの長さ(バイト)
  • レスポンスで受信したcookie
  • 設定されたmatch grepアイテムの結果
  • 設定されたextract grepアイテムで抽出されたデータ
  • payload grepが設定されている場合、ペイロードがレスポンス内に返ってきたかどうか
  • 設定されている場合、リダイレクトをたどった数
  • ユーザーが適用したコメント.

任意の列ヘッダーをクリックすると、一覧内のコンテンツの並び替えができます(ヘッダーをクリックすると、昇順・降順・ソートなしを循環します)。例えば、最新のアイテムが一覧のトップに来て"上方"にリストが延びるのが好きな場合は、リクエスト番号列を降順に並べ替えます。

プロフェッショナル版 Ctrlキーを押しながらヘッダーをクリックすると、列のコンテンツをコピーできます。

一覧内のアイテムを選択すると、リクエストとレスポンスの保存の設定で設定したように、そのリクエストとレスポンスがリクエスト/レスポンス部に表示されます。リダイレクトをたどるよう設定していると、最初のリクエストと最後のレスポンスと共に、途中のレスポンスとリクエストも表示されます。リクエスト/レスポンス部には、各メッセージのHTTPメッセージエディターがあり、詳細な分析ができます。一覧でアイテムをダブルクリックすると、ポップアップウィンドウでリクエストとレスポンスの表示ができます。

結果の分析

攻撃結果を効率的に解釈するには、気になるまたは成功したサーバーレスポンスをみつけ、これらを生成したリクエストの特定が重要です。気になるレスポンスは通常、次のいずれかの方法で区別できます:

  • 異なるHTTPステータスコード
  • 異なるレスポンス長
  • 特定の表現の有無
  • エラーまたはタイムアウトの発生
  • レスポンスの受信または完了までにかかる時間

例えば、コンテンツ探索をしている際、存在しているリソースへのリクエストには様々なサイズの"200 OK"レスポンスが返り、存在しないリソースへのリクエストには"404 Not found"レスポンスや、固定長のカスタムエラーページが"200 OK"で返ってくるかもしれません。あるいはパスワード推測攻撃では、ログイン試行が失敗した場合は"login failed"というキーワードが含まれた"200 OK"レスポンスで、ログイン試行が成功した場合は"302 Object moved"レスポンスや、"welcome"と書かれた異なるサイズの"200 OK"が返ってくるかもしれません。一般的に、レスポンス形式の種類は、実行された攻撃の種類に依存します。

次のテクニックで、攻撃結果の分析と気になるアイテムの特定を助けます:

  • 順番に各列のヘッダーをクリックし、並べ替えられた一覧をレビューして異常なアイテムを特定します。
  • 追加の列を再表示し、応答時間とcookieを分析します。
  • ディスプレイフィルターを使って、特定の文字列を含む、または含まないアイテムを除去します。
  • 一部のレスポンスで見つかった文字列を含むレスポンスを特定するために、新たなmatch grepアイテムを作り、新たに追加した列でソートします。
  • エラーメッセージや気になるデータがレスポンスに現れていた場合、このデータを抽出するため新たなextract grepを作り、新たに追加した列でソートします。

ディスプレイフィルター

結果タブのディスプレイフィルターを使ってビューから一部のコンテンツを非表示にすると、注目するコンテンツの解析や作業が容易になります。

一覧の上部のフィルターバーには、現在のディスプレイフィルターについての説明があります。フィルターバーをクリックすると、編集用にフィルターオプションが開きます。フィルターは、次の属性について設定できます:

  • プロフェッショナル版 検索語 - 指定された検索語がレスポンスに含まれている、または含まれていないかによってフィルターできます。検索語は、リテラル文字列または正規表現で設定でき、大文字小文字を区別するかどうかも設定できます。"一致しないものを検索"オプションを選択すると、検索語が存在しないアイテムのみにマッチします。
  • ステータスコード - 様々なHTTPステータスコードで表示または非表示を設定できます。
  • 注釈 - ユーザーが指定したコメントや色付けがあるアイテムだけを表示するよう設定できます。

結果一覧に表示されるコンテンツは、データベースを反映したビューで、ディスプレイフィルターはビューへの表示内容を制御します。一部のアイテムを隠すようフィルターをセットすると、それらは非表示になるだけで削除されるわけではなく、関連するフィルターを解除すれば再度表示されます。フィルターを使用すると、機械的に生成された大量の結果セット(例えば多くのパラメーターを含むファジングによるものなど)の中から、異なる種類の特徴的なレスポンスが現れるものを調べるのが楽になります。

注釈

攻撃の結果アイテムへのコメントの追加や色付けにより、注釈が付けられます。後で深掘りするレスポンスにフラグを立てておくのに便利です。

色付けは、2 つの方法で追加できます:

  • 一番左の列のドロップダウンメニューを使用して、個々のアイテムを色付けできます。
  • コンテキストメニューの"色付け"を使用して、１つまたは複数の選択されたアイテムに色付けできます。

コメントは、2 つの方法で追加できます。

  • 該当するエントリーのコメント列をダブルクリックし、その場で追加または編集できます。
  • コンテキストメニューの"コメントを追加"を使用して、選択された1 つ以上のアイテムにコメントできます。

気になるリクエストに注釈を付けておくと、列のソートやディスプレイフィルターによって、後でこれらのアイテムを素早く探し出せます。

テストワークフロー

攻撃結果では、すべてのリクエストおよびレスポンスの詳細を表示するだけでなく、コンテキストメニューから指定した攻撃を開始したり、その他のアクションの制御などが可能です。次のオプションがあります:

  • スキャン/巡回/送信する ... - 任意のアイテムを他のBurpツールに送信し、さらなる攻撃や解析が実行できます。ツール間のリクエストの送信機能は、Burpのユーザー主導ワークフローの中核をなします。 
  • レスポンスをブラウザで表示 - 選択したレスポンスをブラウザ内でレンダリングさせ、Burpの組み込みHTMLレンダラの制限を回避します。(Burpのインスタンスをプロキシーに設定した)ブラウザにペーストできる一意なURLが生成され、レスポンスがレンダリングされます。その結果、ブラウザのリクエストはBurpによって処理され(リクエストは元のWebサーバーには送信されません)、選択したレスポンスを正確に返し、最初にリクエストされたURLのコンテキストでブラウザはレスポンスを処理します。したがって、レスポンス内の相対リンクは、ブラウザによって正しく処理されます。その結果、ブラウザはレスポンスのレンダリングの課程で追加のリクエスト(画像やCSSなど)を生成し、これらはBurpの通常の方法で処理されます。
  • ブラウザでリクエスト - 選択したリクエストを(Burpのインスタンスをプロキシーに設定した)ブラウザで再送信できます。次のサブオプションがあります:
    • 元のセッションを使う - 元のリクエストにあるCookieヘッダーを正確に使ったリクエストを発行します。
    • 現在のブラウザのセッションを使う -   ブラウザのcookieを使ったリクエストを発行します。
  • プロフェッショナル版 CSRF PoC作成 - CSRF PoC作成 機能で、ブラウザで表示したとき、選択したリクエストが発行されるHTMLを作成できます。
  • サイトマップに追加 - 選択したアイテムをBurpのTargetサイトマップに追加します。Intruderをカスタマイズしたコンテンツ探索に使っていて、他の方法では発見されずサイトマップに追加されていない新たなサーバーリソースが発見された場合に、このオプションが便利です。
  • アイテムを再リクエスト - 選択したアイテムが攻撃エンジンによって再リクエストされるようキューに入れます。アイテムが再リクエストされると、アイテムのエントリーや関連するHTTPメッセージが、新たなリクエストを元に更新されます。このオプションは、一部の攻撃リクエストがネットワークエラーで失敗した場合や、断続的な問題により異常なレスポンスを受信していた場合に便利です。ベースリクエストやその他の関連する設定を変更した場合に、元の設定で再リクエストしたい場合にも使用できます。
  • レスポンスから抽出grepを定義 - レスポンス抽出ルールダイアログが開き、選択されたレスポンスのアイテムが表示され、そのレスポンスを元に新たなextract grepアイテムを作成できます。一部の攻撃リクエストが、元のリクエストと異なるタイプのレスポンスを生成する場合(例えば、ファジングの際の特定フォーマットのエラーメッセージや、クレデンシャルの推測の際の異なるログインメッセージなど)に便利です。このような異なるレスポンスが発生した場合、レスポンスの気になる部分を抽出する新たなextract grepアイテムを作成するのに便利で、他の同様のレスポンスのコンテンツを素早くレビューできます。
  • コメントを追加 - 選択したアイテムにコメントを追加できます。詳細は、 注釈を参照してください。
  • 色付け - 選択したアイテムを色付けします。詳細は、 注釈を参照してください。
  • アイテムを削除 - 選択したアイテムを完全に削除します。
  • リンクをコピー - 選択したアイテムのリンクを解析し、クリップボードにコピーします。
  • アイテムを保存 - 完全なリクエスト・レスポンス・レスポンス長・HTTPステータスコード・MIMEタイプなど全ての関連するメタデータを含む、選択したアイテムに関するデータを、指定したファイルにXMLフォーマットで保存します。

攻撃設定タブ

結果タブと同様に、攻撃ウィンドウには現在の攻撃の基になったメインのUIでの各設定タブが複製されます。攻撃が進行中、攻撃設定を確認して変更できます。

詳細については、各設定タブのヘルプを参照してください。

  • Target
  • ポジション
  • ペイロード
  • オプション

実行中の攻撃の設定を変更する場合、次の点に注意してください。

  • 攻撃設定のうち、一部の攻撃設定(攻撃タイプやペイロードの種類など)は重要で、攻撃が開始された後は変更できません。
  • 攻撃の実行中に設定の一部を変更すると、予期しない効果が起こる場合があります。例えば、数値ペイロードを使っているときにToフィールドを変更すると、実行中の各キーに対して変更が影響します; もし、Toフィールドの数字を最初に削除すると、その瞬間Toフィールドは今より小さい数字になるので攻撃が突然終了する可能性があります。設定を変更する前に、実行中の攻撃の一時停止を強くお勧めします。

結果メニュー

結果ビューには、攻撃の制御や、他のアクションを実行するためのコマンドといくつかのメニューがあります。これらを以下に記載します。

攻撃メニュー

攻撃の一時停止、再開、再送するコマンドがあります。

保存メニュー

  • 攻撃 - 結果を含む、現在の攻撃のコピーを保存します。保存したファイルは、BurpメインUIのIntruderメニューから"保存した攻撃を開く"を使用すると、再度読み込めます。
  • 結果テーブル - 結果一覧をテキストファイルに保存します。全ての行を保存するか、選択した行のみ保存するか、選択できます。含める列、及び列の区切り文字も選択できます。さらなる分析のために結果をスプレッドシートにエクスポートする場合や、単一の列(extract grep機能で抽出したデータなど)を他の攻撃やツールの入力ファイルとして使う場合に、便利です。
  • サーバーレスポンス - 全てのリクエストに対して受信した全てのレスポンスを保存します。個々のファイル(連番)か、順番に連結した1つのファイルのいずれかに保存できます。
  • 攻撃設定 - 現在実行中の攻撃の設定(結果ではなく)を保存します。BurpメインUIのIntruderメニューから"攻撃設定を読み込み"を使用すると、攻撃設定を再度読み込めます。

列メニュー

攻撃の結果一覧に表示する列を選択します。
