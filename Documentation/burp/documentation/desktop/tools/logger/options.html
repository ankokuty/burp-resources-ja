Loggerオプション

Loggerは、キャプチャするものと表示するものをそれぞれ設定できるよう個別のコントロールがあります。特定の何かを探す場合に、Loggerがキャプチャや表示するものを絞り込めます。

キャプチャオプション

"キャプチャフィルタ"バーをクリックすると、キャプチャするレコードの設定画面が開きます。

キャプチャ制限

レコードを保持するためにLoggerに割り当てるメモリ量を調整します。デフォルト50MB、またはBurp Suiteに1GB以上のメモリを割り当てた場合は100MBです。ドロップダウンメニューで制限を変更するか、特定のメモリ量を入力できます。パフォーマンスの問題を回避するために、大量のレコードが必要になる特定の問題がある場合にのみ、Loggerに割り当てられるメモリの量を増やしてください。

リクエストタイプによるキャプチャ

スコープ内のレコードのみをキャプチャ、応答のないアイテムを破棄、パラメータを含むリクエストのみをキャプチャする設定ができます。

MIMEタイプによるキャプチャ

HTMLやXMLなど、LoggerがキャプチャするMIMEタイプを選択できます。

ステータスコードによるキャプチャ

Loggerがキャプチャするステータスコードを選択できます。

ツールによるキャプチャ

Loggerがレコードをキャプチャするツールを選択できます。デフォルトで、Loggerは全ツールからのレコードをキャプチャします。このコントロールを使用すると、ノイズの多いツールのレコードを破棄したり、1つのツールのレコードのみをキャプチャなどができます。

検索語でキャプチャ

検索ボックスに何か指定されている場合、レコードに検索語が含まれている場合にのみレコードをキャプチャします。(リテラル文字列ではなく)正規表現で検索するか、および検索で大文字と小文字を区別するか、一致しないものを検索(検索語を含むレコードを破棄)するかを制御するチェックボックスがあります。

セッションハンドリング

このチェックボックスを使用すると、Loggerはセッション処理要求を破棄します。

リクエスト/レスポンスのサイズ制限

これらのコントロールを使用すると、Loggerがキャプチャするリクエストやレスポンスのサイズを制限できます。それぞれのデフォルトは1MBです。

表示オプション

レコードがキャプチャされると、表示できるようになります。ただし、キャプチャされたレコードのどれを表示するかは、いつでもフィルタリングできます。"表示フィルタ"バーをクリックすると、表示するレコードをの設定画面が開きます。

リクエストタイプでフィルタ

スコープ内のレコードのみを表示、応答のないアイテムを破棄、パラメータを含むリクエストのみを表示する設定ができます。

MIMEタイプでフィルタ

HTMLやXMLなど、Loggerが表示するMIMEタイプを選択できます。

ステータスコードでフィルタ

Loggerが表示するステータスコードを選択できます。

ツールでフィルタ

Loggerがレコードを表示するツールを選択できます。デフォルトで、Loggerは全ツールからのレコードを表示します。このコントロールを使用すると、ノイズの多いツールのレコードを非表示にしたり、1つのツールのレコードのみを表示などができます。

検索語でフィルタ

検索ボックスに何か指定されている場合、レコードに検索語が含まれている場合にのみレコードを表示します。(リテラル文字列ではなく)正規表現で検索するか、および検索で大文字と小文字を区別するか、一致しないものを検索(検索語を含むレコードを非表示)するかを制御するチェックボックスがあります。

拡張子でフィルタ

ファイル拡張子に基づいてレコードを表示または非表示にできます。

注釈でフィルタ

レコードへのコメントや色づけなど注釈が付けられたレコードのみを表示できます。
