Logger入門

Loggerタブには、Burpの全ツールと対象Webサイトの間で送受信されたすべてのHTTPトラフィックのログが表示されます。これには、Burp Scannerのような自動的にリクエストを送信するツールや拡張も含まれます。

ここから、次のことが可能です:

  • Burpのツールや拡張から送信されたリクエストの調査。
  • Burp Scannerが送信したリクエストをリアルタイムで見る。
  • 拡張の動作を検証する。

LoggerとProxy HTTP履歴の違いは？

Proxy > HTTP履歴タブには、Burpブラウザからのトラフィックのみが表示されます。Loggerはデフォルトで、Burp Suiteのすべてのツールからのトラフィックを表示します。

Loggerタブを使い始めるには、下記のチュートリアルを推奨します。

詳細は、完全なドキュメントを参照してください。

チュートリアル

このチュートリアルでは、Loggerタブを使用して、Burp Scannerが送信したリクエストと対象Webサイトからのレスポンス結果を表示する方法を説明します。

ステップ1: ラボへアクセスする

BurpブラウザとLoggerタブを並べて開きます。

Burpブラウザで次のURLにアクセスします:

https://portswigger.net/web-security/sql-injection/lab-retrieve-hidden-data

Access the labをクリックしプロンプトが表示されたら、自分のPortSwiggerアカウントでログインします。すると、意図的に脆弱性を作り込んだあなた専用のショッピングサイトが開きます。

ステップ2: Loggerタブでリクエストを表示する

Loggerタブには、Proxy > HTTP履歴タブと同様に、ブラウザが行った各リクエストの詳細がリアルタイムで表示されます。

ショッピングサイトで、カテゴリーフィルタをどれか適用します。するとクエリパラメータ付きのリクエストが送信されることに注目してください。

ステップ3: Burp Scannerで特定のリクエストを診断する

Loggerタブからそれぞれのリクエストを選択して、Burp Scannerで診断ができます。

categoryクエリパラメータが付いたリクエストを右クリックし、アクティブスキャン実行を選択します。

ステップ4: Burp Scannerが作成したリクエストを検証する

Burp Scannerは、このリクエストのみを使用して診断を実行します。

Proxy履歴とは異なりLoggerタブには、Burp Scannerが診断する際に生成したリクエストも表示されていることに注目してください。

これはTool列で確認できます。

任意のリクエストを選択すると、メッセージエディタに表示されます。

注

ダッシュボードのタスクにも、そのタスク専用のLoggerタブがあります。詳細表示をクリックすると表示されます。

ステップ5: Loggerタブをソートとフィルタリングする

ログはソートやフィルタリングもできます。

ログをソートするには、列ヘッダをクリックします。ログをソートして、最新のリクエストが上に来るようにすると便利です。

表示フィルタバーをクリックすると、表示するリクエストをフィルタリングできます。

ログのフィルタリングだけではなく、キャプチャフィルタバーをクリックすると、そもそもどのリクエストをログに記録するかを制御できます。これらの設定では、ログが使用できるメモリ量や、リクエストとレスポンスが保存される最大サイズも制御します。

ステップ6: Logger履歴を無効にして消去する

すべてのアイテムのログを無効にするには、ログ: オン/オフボタンをクリックします。

画面右上のログ: オン/オフボタンの下にあるゴミ箱のアイコンをクリックすると、ログを完全に消去できます。クリアしてしまうと、ログは復旧できません。
