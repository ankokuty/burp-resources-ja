Burp Proxyのインターセプト機能入門

Burp Proxyのインターセプト機能は、ブラウザと対象サーバ間で送受信されるHTTPリクエストとレスポンスをインターセプトします。これにより、さまざまな操作によるWebサイトの挙動の変化を調査できます。

インターセプト機能を使うと、次のような便利なアクションが行えます:

  • リクエストをインターセプトし、サーバに転送する前に変更する。
  • 興味深いリクエストを、RepeaterやIntruderなどBurpの他のツールに送信し、さらにテストする。
  • リクエストがサーバに送信されないよう破棄する。

インターセプト機能は、Proxy > インターセプトタブにあります。

Burp Proxyのインターセプト機能の使い方は、次のチュートーリアルを推奨します。

詳細は、完全なドキュメントを参照してください。

チュートリアル

このチュートリアルでは、Burp Proxyを使ってリクエストのインターセプトと変更を行い、サーバの反応を確認します。

ステップ1: ラボへアクセスする

Burpブラウザを開き、次のURLにアクセスします:

https://portswigger.net/web-security/logic-flaws/examples/lab-logic-flaws-excessive-trust-in-client-side-controls

Access the labをクリックしプロンプトが表示されたら、自分のPortSwiggerアカウントでログインします。すると、意図的に脆弱性を作り込んだあなた専用のショッピングサイトが開きます。

ステップ2: ユーザアカウントでログインする

My accountをクリックし、次の認証情報でログインします: wiener:peter。

ステップ3: 買いたいものを探す

Homeをクリックし、Lightweight "l33t" leather jacketの詳細を確認します。

ステップ4: カートへ追加するリクエストをインターセプトする

Burpで、Proxy > インターセプトタブに移動します。

インターセプト有効になっていることを確認します。

ブラウザで、レザージャケットをカートに追加します。

Burpで、POST /cartリクエストがインターセプトされていることに注目してください。

注

ブラウザがバックグラウンドで何か他のことをしていると、最初にProxy > インターセプトタブに別のリクエストが表示されることがあります。この場合は、上のスクリーンショットにあるようにPOST /cartリクエストが表示されるまで転送をクリックします。

ステップ5: リクエストを変更する

リクエストボディで、priceパラメータの値を1に変更します。

ステップ6: 変更したリクエストをサーバに送信する

転送をクリックし、変更したリクエストをサーバに送信します。

インターセプトを再び無効に切り替えます。その後のリクエストはBurp Proxyを素通りするため、通常通りサイトのブラウズができます。ブラウザが発行したリクエストは引き続き、Proxy > HTTP履歴タブに記録されます。

Burpブラウザで、右上のバスケットアイコンをクリックすると、カートが表示されます。ジャケットが追加されていますが、価格はわずか1セントに変更されており、購入の完了までできることに注目してください。

注

Webインターフェースだけでは価格は変更できません。この変更ができたのは、Burp Proxyのおかげです。

注

同じリクエストで異なる入力をテストしたい場合は、Burp Repeaterにリクエストを送信できます。これは、同じリクエストを何度でも編集して再送信できるので、その都度インターセプトする必要がありません。詳細は、Burp Repeater入門を参照してください。

また、Webセキュリティアカデミーにある意図的に脆弱性を作り込んだ"ラボ"サイトを使って、Proxyインターセプトやその他のBurp Suite機能の使い方を練習できます。
