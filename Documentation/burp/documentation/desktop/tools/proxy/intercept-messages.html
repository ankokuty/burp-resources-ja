メッセージのインターセプト

Proxy > インターセプトタブで、ブラウザと対象サーバ間で送受信されるHTTPリクエストとレスポンスをインターセプトします。これにより、Webサイトを操作したときの挙動の変化を調査できます。

インターセプト機能を使うと、次のような便利なアクションが行えます:

  • リクエストをインターセプトし、サーバに転送する前に変更する。
  • 興味深いリクエストを、RepeaterやIntruderなどBurpの他のツールに送信し、さらにテストする。
  • リクエストがサーバに送信されないよう破棄する。

入門

メッセージのインターセプトについて手っ取り早く知りたい場合は、入門セクションのチュートリアルを参照してください:

  • HTTPトラフィックのインターセプト。
  • HTTPトラフィックの変更。

コントロール

メッセージをインターセプトすると、リクエストの詳細がProxy > インターセプトタブに入力されます。パネルの上部に対象サーバの詳細が表示されます。HTTPリクエストの場合、対象サーバを手動で編集できます。それには、対象を編集メニューを選択してください。

パネルには、次のコントロールもあります:

  • 転送 - メッセージの確認や編集をした後、対象宛てにメッセージを送信するには転送をクリックします。
  • 破棄 - リクエストが対象宛てに届かないようキャンセルするには、破棄をクリックします。

  • インターセプト有効/無効 - すべてのインターセプトの有効と無効を切り替えます:

    • ボタンにインターセプト有効と表示されている場合、メッセージがインターセプトされます。また、HTTPやWebSocketメッセージのインターセプトオプションで、メッセージを自動的に転送するように設定もできます。
    • ボタンにインターセプト無効と表示されている場合、すべてのメッセージが自動的に転送されます。
  • アクション - メインパネルのコンテキストメニューが表示されます。ここから、スキャンの実行や他のBurpツールへのリクエスト送信など、さまざまなアクションを実行できます。

  • コメント欄 - 気になるアイテムにコメントを追加でき、後で特定しやすくできます。入力したコメントは、HTTP履歴のアイテムにも表示されます。

    • HTTPリクエストにコメントを追加した場合、そのレスポンスがインターセプトされると、そのコメントが再度表示されます。
  • 色付け - 気になるアイテムに色付けして強調表示できます。色付けはコメントと同じように動作し、Proxy履歴やインターセプトしたレスポンスに表示されます。

注

インターセプトしたメッセージは、ホットキーで転送や破棄ができます。デフォルトでは、Ctrl+Fで現在のメッセージを転送します。ホットキーオプションでホットキーのデフォルトを変更できます。

メッセージ表示

メインパネルのメッセージエディタには、現在インターセプトしているメッセージが表示されます。ここから、メッセージを分析したり、それに対するアクションを実行できます。

アクションをクリックするとコンテキストメニューが表示され、標準機能にアクセスできます。HTTPメッセージに対して、次の操作も行えます:

  • リクエスト/レスポンスをインターセプトしない - ホスト名、ファイル拡張子、HTTPステータスコードなど指定した特徴のメッセージを自動的に転送するように、インターセプトルールを追加できます。この機能は、興味のないリクエストや特定のタイプのレスポンスが多い場合に使用します。
  • インターセプト実行 - このオプションを選択すると、現在表示されているリクエストに対するレスポンスがインターセプトされます。このオプションは、リクエストにのみ使用できます。

プロトコル

Inspectorを使用して、リクエストのプロトコルを編集できます。詳細は、HTTP/2ドキュメントを参照してください。
