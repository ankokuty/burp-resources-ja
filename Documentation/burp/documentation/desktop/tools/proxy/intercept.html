メッセージのインターセプト

インターセプトタブは、ブラウザとWebサーバ間を通過するHTTPとWebSocketメッセージの表示や変更に使われます。すべてのメッセージを表示、インターセプト、変更できるこの機能は、Burpのユーザ主導型ワークフローの中核です。Burp Proxyオプションのインターセプトルールで、どのHTTPリクエストとレスポンスをインターセプトするか(たとえば、スコープ内のアイテム、指定した拡張子のアイテム、パラメータを含むリクエストなど)を設定できます。どのWebSocketメッセージをインターセプトするかも、設定できます。

コントロール

インターセプトしたメッセージが表示されているパネルの上部に、宛先サーバの詳細が表示されます。HTTPリクエストについて、サーバが表示されている部分かその横のボタンをクリックすると、リクエストを送信するサーバを手動で編集できます。

パネルには、次のコントロールもあります:

  • 転送 - メッセージをレビューし(必要なら)編集して転送をクリックすると、サーバまたはブラウザにメッセージが送信されます。
  • 破棄 - メッセージを破棄する場合に使用し、メッセージは転送されません。
  • インターセプト有効/無効 - すべてのインターセプトの有効と無効を切り替えます。ボタンにインターセプト有効と表示されていれば、メッセージはインターセプトされるか、HTTPとWebSocketメッセージのインターセプト設定に基づいて自動的に転送されます。ボタンにインターセプトオフと表示されている場合、すべてのメッセージが自動的に転送されます。
  • アクション - 現在表示されているメッセージに対して実行できるアクションのメニューが表示されます。これらは、インターセプトしたメッセージの表示のコンテキストメニューに表示されるオプションと同じです。
  • コメントフィールド - 気になるアイテムにコメントを追加でき、後で特定しやすくなります。インターセプトパネルで追加したコメントは、Proxy履歴の関連アイテムに表示されます。さらに、HTTPリクエストにコメントを追加した場合、そのコメントはレスポンスがインターセプトされた場合に再度表示されます。
  • 色付け - 気になるアイテムに色付けして強調表示できます。コメント同様、色付けもProxy履歴とインターセプトしたレスポンスに表示されます。

注

メッセージの転送や破棄にホットキーも使えます。デフォルトで、Ctrl+Fで現在のメッセージを転送します。ホットキーオプションでホットキーのデフォルトを変更できます。

メッセージ表示

インターセプトタブのメインパネルには、現在インターセプトしたメッセージを表示するメッセージエディタがあり、メッセージの分析やさまざまなアクションを実行できます。

エディタのコンテキストメニューには、多数の有用なアイテムがあります。エディタ自体の提供する標準的な機能に加え、HTTPメッセージに対して次のアクションが可能です:

  • インターセプトしないリクエスト/レスポンス - 現在表示されているメッセージと共通する(ホスト名、ファイル拡張子、HTTPステータスコードなどに基づく)メッセージを以後インターセプトしないように、これらのコマンドで素早くインターセプトルールを設定できます。特定タイプの関心のないリクエストやレスポンスをインターセプトしてしまっている場合、このオプションでそのようなメッセージを自動的に転送できます。
  • インターセプト実行 - リクエストにのみ有効で、現在表示されているリクエストに対するレスポンスをインターセプトします。

プロトコル

Inspectorがリクエストで使用するプロトコルを表示や編集できます。詳細は、HTTP/2ドキュメントを参照してください。
