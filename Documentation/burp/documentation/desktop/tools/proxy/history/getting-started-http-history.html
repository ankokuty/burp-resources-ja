Burp ProxyのHTTP履歴入門

Burp Proxyは、BurpブラウザとWebサーバ間の通信を表示、インターセプト、さらに変更もできるWebプロキシサーバです。

HTTP履歴タブには、Burpブラウザが行ったHTTPリクエストと、それに対応するサーバからのレスポンスのログが表示されます。

これにより次が可能です:

  • 対象Webサイトの挙動を調査する。
  • 有用なパラメータやその他の入力を含んでいるHTTPリクエストを探す。
  • HTTPレスポンスの中から、脆弱性を示す特徴を探す。
  • 興味深いリクエストをBurp Suiteの他のツールに送信し、さらにテストする。

HTTP履歴タブを使い始めるには、下記のチュートリアルを推奨します。

HTTP履歴タブの機能の詳細は、完全なドキュメントを参照してください。

チュートリアル

このチュートリアルでは、Webセキュリティアカデミーにある意図的に脆弱性を作り込んだラボを使用して、HTTP履歴タブの主な機能を学びます。

ステップ1: ラボへアクセスする

Burpブラウザを開き、次のラボにアクセスします:

https://portswigger.net/web-security/sql-injection/union-attacks/lab-determine-number-of-columns

Access the labをクリックしプロンプトが表示されたら、自分のPortSwiggerアカウントでログインします。すると、意図的に脆弱性を作り込んだあなた専用のショッピングサイトが開きます。

ステップ2: HTTP履歴を入力する

HTTP履歴タブがどのように機能するか確認するために、まずリクエストをタブに入力する必要があります。

BurpブラウザとBurp Suiteを並べて配置します。

ショッピングサイトで、いくつかの商品ページを参照します。このときHTTP履歴タブに、ブラウザが行った各リクエストの詳細がリアルタイムに表示されることに注目してください。

続きを読む

履歴一覧

ステップ3: リクエストとレスポンスを表示する

特定のリクエストの詳細を表示するには、履歴一覧でそのエントリをクリックします。すると、リクエストとレスポンスの全文がメッセージエディタに表示されます。

メッセージエディタには、リクエストとレスポンスを表示するためのさまざまなビューがあります。リクエストとレスポンスの上にあるボタンで切り替えられます。

ウィンドウの右側には、Inspectorパネルがあります。Inspectorは、メッセージの解析や作業を容易にするツールです。

続きを読む

  • メッセージエディタ
  • Inspector

ステップ4: 履歴一覧をソートとフィルタリングする

履歴一覧は、ソートやフィルタリングもできます。

特定の列で一覧をソートするには、列ヘッダをクリックするだけです。最新のリクエストが履歴一覧の上に来るようにソートすると便利です。

フィルタ設定を開くには、履歴一覧の上にあるフィルタバーをクリックします。

ステップ5: 他のツールにリクエストを送信する

履歴一覧やメッセージエディタでリクエストを右クリックすると、コンテキストメニューが表示されます。ここから、さまざまなアクションを選択できます。

最もよく使われるアクションの1つは、興味深いリクエストをさらにテストするために、Burpの他のツールに送ることです。

Burp ProxyのHTTP履歴の詳細

これで、HTTP履歴タブを使用してリクエストを調査し、他のツールに送信する概要を学びました。

Proxy履歴でのリクエストに対する操作は、他にもあります:

  • リクエストに色づけやコメントを追加する。
  • 特定のエンドポイントをテストスコープに追加または削除する。
  • 元のセッションまたは新しいセッションで、ブラウザを使って再度リクエストを送信する。
  • リクエストに対するレスポンスをBurpブラウザでレンダリングして表示する。

HTTP履歴とメッセージエディタの詳細は、次を参照してください:

  • HTTP履歴タブの完全なドキュメント。
  • メッセージエディタの完全なドキュメント。

HTTP履歴タブの使い方をもっと練習したい場合は、Webセキュリティアカデミーに挑戦してください。
