Burp ProxyのWebSocket入門

Burp Proxyは、BurpブラウザとWebサーバ間の通信を表示、インターセプト、さらに変更もできるWebプロキシサーバです。

WebSocket履歴タブには、BurpブラウザがWebサーバと送受信したWebSocketメッセージのログが表示されます。

これにより次が可能です:

  • 対象Webサイトの挙動を調査する。
  • WebSocketのハンドシェイクやメッセージに含まれる脆弱性を探す。
  • 興味深いメッセージをBurp Suiteの他のツールに送信し、さらにテストする。

WebSocket履歴タブを使い始めるには、下記のチュートリアルを推奨します。

WebSocket履歴タブの詳細は、完全なドキュメントを参照してください。

チュートリアル

このチュートリアルでは、Webセキュリティアカデミーにある意図的に脆弱性を作り込んだラボを使用して、WebSocket履歴の基本を学びます。

ステップ1: ラボへアクセスする

Burpブラウザを開き、次のラボにアクセスします:

https://portswigger.net/web-security/websockets/lab-manipulating-messages-to-exploit-vulnerabilities

Access the labをクリックしプロンプトが表示されたら、自分のPortSwiggerアカウントでログインします。すると、意図的に脆弱性を作り込んだあなた専用のショッピングサイトが開きます。ライブチャット機能もあります。

ステップ2: WebSocket履歴を入力する

WebSocket履歴タブがどのように機能するか確認するために、まずメッセージをタブに入力する必要があります。

BurpブラウザとBurp Suiteを並べて配置します。

ショッピングサイトで、Live chatをクリックします。

チャットボックスでメッセージを送信すると、行った操作がWebSocket履歴に反映されることを確認してください。

履歴一覧

ステップ3: WebSocketメッセージを表示する

特定のWebSocketメッセージの詳細を表示するには、履歴一覧のエントリをクリックします。すると、メッセージの生テキストがメッセージエディタに表示されます。

サーバに送信された最初のハンドシェイクメッセージと、それに続くクライアントとサーバ間のメッセージのやりとりに注目してください。

ステップ4: メッセージ履歴一覧をソートとフィルタリングする

履歴一覧は、ソートやフィルタリングもできます。

特定の列で一覧をソートするには、列ヘッダをクリックするだけです。最新のメッセージが履歴一覧の上に来るようにソートすると便利です。

フィルタ設定を開くには、履歴一覧の上にあるフィルタバーをクリックします。

ステップ5: 他のツールにメッセージを送信する

履歴一覧やメッセージエディタでリクエストを右クリックすると、コンテキストメニューが表示されます。ここから、さまざまなアクションを選択できます。

最もよく使われるアクションの1つは、興味深いメッセージをさらにテストするために、Burpの他のツールに送ることです。

Burp Repeaterを使用して、WebSocketメッセージを編集して再送信し、サーバがどのように反応するかをテストできます。
