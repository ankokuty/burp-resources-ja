Burp Repeaterオプション

Repeaterメニューは、Burp Repeaterの挙動を制御します。次のオプションがあります:

  • Content-Lengthを更新 - 必要に応じてリクエストのContent-Lengthヘッダを自動的に更新します。リクエストにボディが含まれている場合、通常このオプションが不可欠です。
  • gzip/deflateを展開する - レスポンスで受け取ったgzipやdeflateで圧縮されたコンテンツを自動的に展開します。
  • リダイレクトをたどる - 自動でリダイレクトするかどうか設定します。注: 自動的にリダイレクトしないURLへのリダイレクトレスポンスを受け取った場合、UIの上部に"リダイレクトをたどる"ボタンが表示されます。これで、手動でリダイレクト先にアクセスできます。この機能は、各リクエストとレスポンスのリダイレクトを順番にたどる場合に便利です。下記の"リダイレクト中にcookieを処理する"オプションがセットされていると、これらの手動リダイレクトでも新しいcookieが処理されます。
  • リダイレクト中にcookieを処理する - リダイレクトレスポンスでセットされたcookieを、リダイレクト先にアクセスする際送信します。
  • 表示 - リクエスト/レスポンスパネルのレイアウトを設定できます。上下分割、左右分割、またはタブで、HTTPメッセージを表示できます。
  • アクション - リクエストとレスポンスのメッセージエディタのコンテキストメニューと同じオプションが含まれます。
