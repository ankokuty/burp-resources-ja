Burp Repeater入門

Burp Repeaterは、興味深いリクエストを何度でも再発行できます。すると、リクエストのインターセプトを何度もしなくても、さまざな入力に対する対象Webサイトの反応を調べられます。

Burp Repeaterを使用するには、Burpの他のツールのいずれかからリクエストを送信してください。Burp Repeaterにより、例えば脆弱性の調査や、Burp Scannerが発見した脆弱性の確認などが非常に簡単になります。

Burp Repeater使い方は、下記のチュートリアルを推奨します。

詳細は、完全なドキュメントを参照してください。

チュートリアル

この例では、Burp ProxyのHTTP履歴からBurp Repeaterにリクエストを送信してみます。次に、Burp Repeaterを使って、さまざまな入力値のリクエストを再発行し、それがレスポンスにどのような影響を与えるか見てみましょう。

ステップ1: ラボへアクセスする

Burpブラウザを開き、次のURLにアクセスします:

https://portswigger.net/web-security/information-disclosure/exploiting/lab-infoleak-in-error-messages

Access the labをクリックしプロンプトが表示されたら、自分のPortSwiggerアカウントでログインします。すると、意図的に脆弱性を作り込んだあなた専用のショッピングサイトが開きます。

ステップ2: 対象サイトをブラウズする

ブラウザで、いくつかの製品ページをクリックしてサイトを探索します。

ステップ3: 興味深いリクエストを見つける

Burpで、Proxy > HTTP履歴タブを開きます。

商品ページにアクセスするたびに、ブラウザはproductIdクエリパラメータを付けて、GET /productリクエストを送信していることに注目してください。

ステップ4: リクエストをBurp Repeaterに送信する

GET /product?productId=[...]リクエストのどれかを右クリックし、Repeaterに送信を選択します。

ステップ5: リクエストを発行しレスポンスを確認する

Repeaterタブに移動します。送信をクリックすると、リクエストが発行され、サーバからのレスポンスが表示されます。

ステップ6: さまざまな入力でリクエストを再発行する

productIdパラメータの数字を変更し、リクエストを再送します。大きめの数字など任意の数字を指定してみて、レスポンスに影響があるかを確認します。

ステップ7: 想定外の入力を送ってみる

サーバは、このproductIdパラメータで整数値を受け取ることを期待しているようです。別のデータ形式をBurp Repeaterで送信し、何が起こるかを見てください。

productIdを数字以外の文字列にした別のリクエストを送信します。

整数ではないproductIdを送信すると、例外が発生するか確認してください。サーバから、スタックトレースを含む冗長なエラーレスポンスが送信され、ここに有用な情報が漏洩している可能性があります。

ステップ8: リクエスト履歴を見る

矢印を使うと、自分が送ったリクエストとそれに対応するレスポンスの履歴をさかのぼって確認できます。また、各矢印の横にあるドロップダウンメニューから、履歴内の特定リクエストにジャンプできます。

これは、以前に送信したリクエストに戻って、特定の入力をさらに調査する場合に便利です。

このようにさまざまな入力をテストする操作は、Burp Suiteを使って手動テスト行う、最も一般的な作業の1つです。
