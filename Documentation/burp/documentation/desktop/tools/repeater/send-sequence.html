リクエストを順番に送信

Repeaterのリクエストを順番に送信機能を使えば、グループ内のすべてのリクエストを1回で送信できます。グループ内のそれぞれのリクエストを同じ接続で送信することも、個別の接続を使用することもできます。

注

Repeaterでタブグループを作成する方法は、タブグループの管理を参照してください。

グループ内のリクエストを送信するには:

  1. グループを作成し、関連するタブを追加します。
  2. 送信ボタンの横にあるドロップダウンの矢印をクリックし、必要に応じてグループを順番に送信(単一接続)またはグループを順番に送信(個別接続)を選択します。
  3. グループ送信をクリックします。

Repeaterは、グループ内に配置された順番で、すべてのタブからのリクエストを送信しようとします。

一連のリクエストをキャンセルするには、リクエストの送信中にグループのタブのいずれかでキャンセルをクリックします。

一連のリクエスト送信の前提条件

グループ内のタブを単一接続で送信するには、グループが次の条件を満たしている必要があります。

  • グループ内のタブはすべて同じターゲットであること。
  • グループ内のタブはすべて同じリクエストプロトコルを使用すること(つまり、すべてHTTP/1を使用するか、すべてHTTP/2を使用するかのどちらかであること)。
  • グループ内にWebSocketメッセージのタブがないこと。
  • グループ内に空のタブがないこと。

一連のリクエストを個別の接続で送信する場合は、異なるターゲットやプロトコルが使用できます。ただし、WebSocketタブや空のタブを含むグループの場合は、個別の接続でも送信できません。
