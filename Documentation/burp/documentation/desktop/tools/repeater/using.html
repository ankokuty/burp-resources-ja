Burp Repeaterの使い方

Burp Repeaterは、HTTPメッセージやWebSocketメッセージを手動で操作や再発行をし、アプリケーションのレスポンスを分析するための、シンプルなツールです。入力ベースの脆弱性をテストするためにパラメータ値を書き換える、ロジックフローをテストするために特定の順序でリクエストを発行する、Burp Scannerが報告したBurp Scanner issueを検証するためにリクエストを再送するなど、さまざまな目的にRepeaterが利用できます。

RepeaterのUIのそれぞれ個別のタブで、複数の異なるメッセージを同時に操作できます。Repeaterにメッセージを送信すると、それぞれが個別の番号付きタブで開かれます。タブヘッダをダブルクリックすると名前を変更できます。

Burp Repeaterの使い方(HTTPメッセージ)

Burp RepeaterでHTTPメッセージを使用するには、Burpの任意の場所でHTTPメッセージを選択し、コンテキストメニューからRepeaterに送信を選択します。Repeaterに新しいリクエストタブが作成され、対象の詳細や関連する詳細をリクエストメッセージエディタに自動的に設定します。または、手動で新規のRepeaterタブを開き、HTTPオプションを選択します。

HTTPメッセージでは、各Repeaterタブに次のアイテムがあります:

  • リクエストの発行とリクエストの履歴をナビゲートするコントロール。
  • リクエストを送信する対象サーバが表示されます - 変更するには対象の詳細をクリックします。
  • 発行するリクエストを含むHTTPメッセージエディタ。何度も繰り返してリクエストを編集し再発行できます。
  • 最後に発行されたリクエストから受信したレスポンスを表示するHTTPメッセージエディタ。

HTTPリクエスト送信

リクエストを送信する準備ができたら、送信ボタンをクリックしサーバに送信します。レスポンスを受信すると、レスポンス長とタイマー(ミリ秒)とともにレスポンスが表示されます。リクエストとレスポンスのメッセージを分析しさらにアクションを実行するために、通常のHTTPメッセージエディタの機能が使えます。

Burpがメッセージの送信に使用するプロトコルを選択もできます。

注

また、Repeaterタブのグループを1回のクリックで送信もできます。Repeaterタブを順番に送信する方法は、リクエストを順番に送信を参照してください。

HTTPリクエスト履歴

Repeaterの各タブは、その内で作られたリクエストのタブ固有のヒストリーを保持します。<と>ボタンをクリックして履歴を後方、前方に移動し、各リクエストとレスポンスを表示できます。ドロップダウンボタンを利用すると、最近の履歴アイテムのリストが表示され、それらに素早く移動できます。履歴の任意の時点で、現在表示されているリクエストを編集して再発行できます。

Burp Repeaterの使い方(WebSocketメッセージ)

Burp RepeaterでWebSocketメッセージを使用するには、Proxy履歴でWebSocketメッセージを選択し、コンテキストメニューからRepeaterに送信を選択します。または、新規のRepeaterタブを開き、WebSocketオプションを選択します。

WebSocketメッセージでは、各Repeaterのタブに次のアイテムがあります:

  • 送信されるWebSocketメッセージを含むメッセージエディタ。何度も繰り返してリメッセージを編集し再発行できます。
  • メッセージを送信するWebSocket接続。
  • 送受信したすべてのメッセージを表示する履歴一覧と、現在選択されているメッセージを表示するメッセージビューア。

WebSocketメッセージの送信

送信するメッセージの編集や、サーバあるいはクライアントに対して送信するか選択できます。クライアントにメッセージを送信するオプションは、Burp Proxy経由で接続が開いている場合のみ有効です。

メッセージを送信する準備ができたら、送信ボタンをクリックしメッセージを送信します。

オプションで、メッセージを送信した後に受信した次のメッセージを、履歴一覧で自動的に選択できます。

WebSocketメッセージ履歴

履歴一覧には、送受信したすべてのメッセージが表示されます。Burp Repeaterで手動で作成されたメッセージは、Repeater列で区別できます。メッセージを選択すると下部のペインに表示できます。

履歴からメッセージを再送信したい場合は、コンテキストメニューの編集して再送信オプションを選択します。これにより左側のメッセージエディタの選択したメッセージが表示され、必要に応じてメッセージを変更し、送信できます。

Repeaterオプション

Burp Repeaterのオプションにより、リクエスト送信時やレスポンス受信時の、Repeaterの挙動を制御できます。すべてのタブに適用されるRepeaterオプションを設定するには、Repeaterメニュー使用します。Repeaterメニューオプションは、LinuxとWindowsではウィンドウの上部に、Mac OSXでは画面の一番上にあります。

注

Repaterのオプションの詳細は、Burp Repaterのオプションページを参照してください。

タブごとのRepeaterオプション設定

必要に応じて、Repeaterメニューで選択したオプションを、個々のタブで上書きできます。

タブ固有のオプションを設定するには:

  1. 設定したいタブを選択します。
  2. 送信ボタンの横にあるアイコンをクリックすると、Repeaterオプションのコンテキストメニューが表示されます。これらのオプションは、グローバルのRepeaterメニューと同じです。
  3. メニューから必要なオプションを選択します。

タブ固有のメニューでオプションを選択した場合、Repeaterはそのタブですべてのグローバルオプションを無視します。たとえば、グローバルRepeaterメニューでリダイレクト中にcookieを処理するを選択し、タブ固有のメニューでHTTP/1接続を再利用するを選択したとします。この場合そのタブでは、グローバルのリダイレクト中にcookieを処理するの設定は無視され、デフォルト設定に戻ります。そのため、タブからリクエストを送信する前に、タブ固有のメニューですべてのオプションが正しく設定されているかを確認する必要があります。

タブのオプションを変更すると、そのタブの設定アイコンが青くなります。タブの設定アイコンをクリックし、コンテキストメニューからグローバルデフォルトを復元を選択すると、タブをグローバルRepeater設定に戻せます。

プロジェクトファイルを使用している場合、開いているタブに設定したタブ固有のオプションは、Burp Suiteを再度開いたときにも保持されます。

Requestタブの管理

リクエストタブは、タブバー上のコントロールで管理できます。

リクエストをゼロから作成

新しいタブを開くには、+アイコンをクリックし、コンテキストメニューからHTTPまたはWebSocketを選択します。

タブ名の変更

タブ名を変更するには、タブヘッダをダブルクリックして、新しいタブ名を入力します。あるいは、タブを右クリックし、タブ名変更を選択します。

タブ表示の切り替え

Burp Repeaterでは、多数のタブを簡単に操作できるように、2種類のタブ表示があります。タブをスクロール可能な1行で表示するか、複数の行に折り返して一度に表示するかを選択できます。

スクロール表示で画面の端からタブがはみ出した場合、オーバーフローメニューアイコンが表示されます。このアイコンをクリックすると、開いているすべてのタブのドロップダウンリストが表示されます。検索バーで特定のタブも探せます。

タブ表示を変更するには、タブを右クリックするか、オプションメニューをクリックしタブ表示オプションを選択します。

タブを閉じる

タブを閉じる方法は複数あります。

  • タブを1つ閉じるには、そのタブの×ボタンをクリックします。タブを右クリックし、タブを閉じるも選択できます。
  • 選択したタブ以外のタブを閉じるには、タブを右クリックし、他のタブを閉じるを選択します。
  • 選択したタブの片側にあるすべてのタブを閉じるには、タブを右クリックし、左側のタブを閉じるまたは右側のタブを閉じるを選択します。
  • 開いているタブをすべて閉じるには、オプションメニューをクリックし、すべてのタブを閉じるを選択します。
  • グループに属するタブで、選択したタブ以外のグループ内のタブを閉じるには、そのタブを右クリックし、グループ内の他のタブを閉じるを選択します。残りのタブはグループの中に残ります。

最後に閉じたタブを再び開くには、任意のタブを右クリックし、閉じたタブを開くを選択します。

注

Repeaterのタブグループ機能では、関連するタブをグループ化し、リクエストのグループを順番に送信できます。

  • Repeaterでタブグループを作成する方法は、タブグループの管理を参照してください。
  • Repeaterタブを順番に送信する方法は、リクエストを順番に送信を参照してください。
