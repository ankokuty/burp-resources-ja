Burp Suiteモバイルアシスタントをインストール

Burp Suiteモバイルアシスタントは、Burp SuiteによるiOSアプリのテストを容易にするツールです。次の主要機能をサポートしています:

  • iOSデバイスのシステム全体のプロキシ設定を変更し、実行中のBurpインスタンスにHTTP(S)トラフィックを容易にリダイレクトできます。
  • 選択されたアプリのSSL証明書ピニングを回避し、Burp SuiteがHTTPS接続を解読し、すべてのトラフィックをインターセプト、調査、変更できるよう試みます。

注

Burp Suiteモバイルアシスタントは機能の性質上、脱獄されたデバイスにインストールする必要があります。脱獄では通常、モバイルアシスタントのインストールに使用できる、人気パッケージマネージャーのCydiaをインストールします。上級ユーザはモバイルアシスタントを直接インストールしたいかもしれません。たとえば、http://[burp-host-ip]:8080/mobileassistant.deb のように、Burp Suiteのブラウザ内インタフェースにブラウザでアクセスすると可能です。Cydiaは厳密には必要ありませんが、モバイルアシスタントはCydia Substrateに依存しています。

このチュートリアルでは、Cydiaを使用してモバイルアシスタントをインストールする方法を示します。

脱獄したデバイスからネットワークアクセスできるシステムでBurp Suiteを起動し、Proxyリスナーがアクセスできるネットワークインタフェースとポートをバインドしているか確認します。

次のステップは、Burp SuiteのインスタンスをCydiaのパッケージソースに追加します。

脱獄したデバイスで、Cydiaを開きます。

Sourcesタブに移動します。

右上隅にあるEditボタンをタップします。

左上隅にあるAddボタンをタップします。

APT URLプロンプトにhttp://[burp-host-ip]:8080のように、プロトコルにはHTTPを使用し、Burp ProxyリスナーのIPアドレス(またはホスト名)とポート番号を入力します。

Cydiaが接続できない場合は、関連するネットワークインタフェースをリッスンするようBurp Proxyリスナーが設定されているか、またはコンピュータのホストファイアウォールや無線ネットワークのデバイス分離によって接続がブロックされていないか確認してください。

終了するには、Doneボタンをタップします。

Burp Suiteが個別のソースとして表示されるはずです。Burp Suite Proオプションをタップします。

Cydiaのパッケージソースから通常の方法でモバイルアシスタントをインストールします。Mobile Assistantを検索するか、新しいソースのパッケージを参照します。

Installをタップします。

Confirmをタップします。ダウンロードが完了したら、Restart Springboardをクリックします。

これで、Mobile Assistantアイコンがデバイスに表示されます。

Burp Suiteを経由したトラフィックのルーティングを開始できます。
