Burp拡張の開発

Javaに精通していれば、Montoya APIを使用して独自のBurp Suite拡張を作成できます。

注

BAppストアにある多くの拡張は、古いExtender APIで開発されています。Burp Suiteは当面の間Extender APIをサポートし続けますが、新しい拡張はMontoya APIを使った開発を強く推奨します。

はじめに、MavenまたはGradleのプロジェクトを新規に作成し、次のようにMontoya APIの依存関係を追加します:

Maven

<dependency> <groupId>net.portswigger.burp.extender</groupId> <artifactId>montoya-api</artifactId> <version>XX.XX.XX</version> </dependency>

Gradle

implementation 'net.portswigger.burp.extender:montoya-api:XX.XX.XX'

注

XX.XX.XXをバージョン番号に置き換えてください。

次に、BurpExtensionインタフェースを実装したクラスを作成します。initialize()メソッドは、MontoyaApiの実装を提供します。これを使用してBurp Suiteの変更や通信ができます。

サポートのために次のようなリソースを提供します:

  • HTTPハンドラやProxyリスナーを登録するような、一般的なタスクの実装例が、Montoya APIのGitHubリポジトリにあります: Montoya API のGitHubリポジトリ。
  • JavaDocリファレンス: Montoya API JavaDoc。
  • 古いExtender APIでのほとんどの機能をMontoya APIで実装する例。各メソッドには、Extender APIの同等のメソッドと同じ名前を付けているので、探しているものを簡単に見つけられます。これは、古い拡張をリファクタリングする場合に特に便利です: 拡張の例。
  • BAppストアで配布されている拡張のフォーク版を含んだ、PortSwiggerのGitHubリポジトリ。しかし、これらのほとんどは古いExtender APIで開発されている点に注意してください。

拡張が完成したら、BAppストアに投稿して他のBurpユーザと共有できます。BAppストアへの拡張の投稿およびBAppsの受入基準を参照してください。

注

拡張から新しいリクエストを発行する場合、HTTP/2フォーマットを使ったケトルなリクエストを自由に送信できます。これにより、HTTP/2専用の脆弱性をテストするための拡張を開発できます。

ただし現在のところ、Burpによって発行されたケトルなリクエストを、拡張からは変更できません。これは、APIを介した場合には正規化されたHTTP/1スタイルの表現にしかアクセスできないためです。
