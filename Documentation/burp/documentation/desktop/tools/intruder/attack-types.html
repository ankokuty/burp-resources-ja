攻撃タイプ

Burp Intruderは、さまざまな攻撃タイプをサポートしています - これらはペイロードポジションにペイロードを割り当てる方法を決定します。リクエストテンプレートエディタ上のドロップダウンを使用して、攻撃タイプを選択できます。次の攻撃タイプがあります。

スナイパー(Sniper)

単一のペイロードセットを使用します。各ペイロードポジションを順番に対象にし、そのポジションに各ペイロードを順番に配置します。リクエストで対象となっていないポジションは影響を受けません - ポジションマーカが削除され、テンプレート上のポジションマーカに囲まれていたテキストは変更されません。この攻撃タイプは、多くのリクエストパラメータに対する一般的な脆弱性のファジングに便利です。攻撃で生成されるリクエストの合計数は、ペイロードポジション数とペイロードセット内のペイロード数の積です。

破城槌(Battering ram)

単一のペイロードセットを使用します。ペイロードをひとつずつ使用し、定義されたペイロードポジションのすべてに、一斉に同じペイロードを配置します。この攻撃タイプは、リクエスト内の複数の箇所(たとえば、cookieと body パラメータ内にあるユーザ名)に同じ値を入れる必要がある攻撃に便利です。攻撃で生成されるリクエストの合計数は、ペイロードセット内のペイロード数です。

熊手(Pitchfork)

複数のペイロードセットを使用します。それぞれのポジション(最大で20個)に対して異なるペイロードを使います。すべてのペイロードセットを同時に使用し、定義されたそれぞれのポジションを1つのペイロードに置き換えます。言い換えると、最初のリクエストでは、1番目のペイロードポジションにペイロードセット1の1番目のペイロードを置き、2番目のペイロードポジションにペイロードセット2の1番目のペイロードを置きます。2つめのリクエストでは、1番目のペイロードポジションにペイロードセット1の2番目のペイロードを置き、2番目のペイロードポジションにペイロードセット2の2番目のペイロードを置きます。この攻撃タイプは、リクエスト内の複数箇所に、関連はあるが異なる値(たとえば、あるパラメータはユーザ名、別のパラメータではユーザ名に対応する既知のID番号)の入力を必要とする場合に便利です。攻撃で生成されるリクエストの合計数は、最小のペイロードセットのペイロード数です。

クラスター爆弾(Cluster bomb)

複数のペイロードセットを使用します。それぞれのポジション(最大で20個)に対して異なるペイロードを使います。攻撃は、ペイロードのすべての組み合わせをテストするように、各ペイロードを反復します。つまり例えば2つのペイロードポジションが存在する場合、ポジション2にペイロードセット2の最初のペイロードを置き、ポジション1にペイロードセット1のすべてのペイロードを繰り返し処理します; 次にポジション2にペイロードセット2の2番目のペイロードを置き、ポジション1にペイロードセット1のすべてのペイロードを繰り返し処理します。この攻撃タイプは、リクエスト内の複数のパラメータで関連性がない場合や、異なる値未知のの入力を必要とする場合に便利です(たとえば認証試行で、あるパラメータがユーザ名で別のパラメータがパスワードの場合などです)。攻撃で生成されるリクエストの合計数は、すべての定義されたペイロードセットのペイロード数の積です - これは非常に大きくなる可能性があります。
