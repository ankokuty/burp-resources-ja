Intruderのリソースプール

リソースプールは、リソースの割り当てを共有するタスクをグループ化したものです。各リソースプールはそれぞれ、同時リクエスト数、またはリクエストの生成頻度、またはその両方を調整できます。Burp ScannerもBurp Intruderもどちらもリソースプールを利用します。

リクエスト間に遅延を入れる場合は、次のオプションから選択できます:

  • 固定の遅延。
  • ランダムに変動する遅延。
  • 遅延を段階的に増加。このオプションは、リクエストが送信されない場合にセッションが有効期限切れになるまでの時間を調べる際に、特に有用です。
  • 自動バックオフ。Burpはデフォルトで、サーバのレート制限に適応するまで、短い遅延をリクエスト間に段階的に追加していきます。これによって通常通り攻撃を行えますが、全体の所要時間が長くなります。必要に応じて、自動バックオフの選択を解除して、この動作を無効にできます。

各タスクは作成時にリソースプールに割り当てられ、いつでもリソースプール間で移動できます。

リソースプールは、異なる速度で自動化リクエストを受け付ける、異なるアプリケーションをテストする場合に特に便利です。また、さまざまなテスト範囲に優先順位を付ける場合も便利です。たとえば、同時リクエスト数を少なくしたタスクを作成しアプリケーション全体のクロールと診断をバックグランドで実行しておき、Intruder攻撃用に別のタスクを作成し、こちらは同時リクエスト数をより多くすれば優先度を上げられます。

進行中または終了した攻撃から、新しいリソースプールは作成できません。攻撃開始後にリソースプールを新たに作成した場合は、ダッシュボードに移動し、歯車設定アイコンをクリックしてプールを作成してください。
