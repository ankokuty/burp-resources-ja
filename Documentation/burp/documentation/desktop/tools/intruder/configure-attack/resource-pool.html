Burp Intruderのリソースプール

リソースプールは、リソースの割り当てを共有するタスクをグループ化したものです。リソースプールを利用すると、次がより簡単になります:

  • 特に複数の攻撃にまたがる、システムリソースの使用状況の管理と優先度付け。
  • 自動化されたリクエストを異なる速度で受け付けるアプリケーションのテスト。

リソースプールの作成

新しいタスクは、デフォルトのリソースプールに割り当てられます。カスタムプールは、攻撃を開始する前にいつでも作成できます。リソースプールを作成するには:

  1. Intruder > リソースプールに移動します。
  2. 新しいリソースプールを作成するを選択します。
  3. プールの名前を入力し、プールの設定を行います。
  4. 攻撃開始時にプールが作成されます。

攻撃開始後に新しいプールを作成する場合は、別のタスクで行うか、Burpのダッシュボードで設定する必要があります。

リソースプールの設定

各リソースプールは、それぞれ独自の速度設定を行えます:

  • 最大同時リクエスト数 - 攻撃で同時に送信するリクエスト数を制限します。これは、対象サーバに過負荷をかけないように、あるいは指定されたレート制限を超えないようにするために有効です。

  • リクエスト間の遅延 - 単位はミリ秒です。遅延の種類は3種類から選択できます:

    • 固定。
    • ランダムに変動。
    • 遅延を段階的に増加 - リクエストが送信されない場合に、セッションが有効期限切れになるまでの時間を決定できます。
  • 自動バックオフ - リクエスト間に短い遅延を自動的に追加し、攻撃がサーバのレート制限に適応するまで段階的に増加させます。これは、デフォルトのリソースプールで有効になっています。

リソースプール間のタスク移動

攻撃を開始する前にタスクを移動するには、Intruder > リソースプールで、プールを選択します。

攻撃中にタスクを移動するには、攻撃ウィンドウのリソースプールタブで、プールを選択します。これにより、システムリソースの使用状況をリアルタイムに管理できます。
