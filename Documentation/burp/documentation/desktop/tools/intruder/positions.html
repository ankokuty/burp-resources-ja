ペイロードポジション

このタブを使用して、ペイロードマーカーと、攻撃の種類(どの位置にペイロードを割り当てるかを決める方法)からなる、攻撃のためのリクエストテンプレートを設定します。

リクエストテンプレート

メインのリクエストエディターを使用して、すべての攻撃の派生元となるリクエストテンプレートを定義します。Burpは攻撃リクエストごとに、リクエストテンプレートから、ペイロードマーカーによって定義されている位置に 1 つまたは複数のペイロードを当てはめます。

リクエストテンプレートを設定する最も簡単な方法は、攻撃したいリクエストをBurpの内のどこかで選び、コンテキストメニューの"Intruderに送信"オプションを選択します。すると選択したリクエストがIntruderの新しいタブに送信され、Targetとポジションタブが自動的に現れます。 

ペイロードマーカー

ペイロードマーカーは、§文字を使用して次のように機能します:

  • マーカーの各ペアは、1 つのペイロードの位置を指定します。
  • 必要に応じて、リクエストテンプレート内の一部のテキストを、マーカーのペアの間に囲めます。
  • ペイロードを当てはめる際、マーカーと囲まれたテキスト両方が、ペイロードに置き換えられます。
  • ペイロードポジションに割り当てられたペイロードがないとき、マーカーは削除されますが、囲われたテキストは変更されません。

設定しやすくするため、Intruderは自動的にペイロードマーカーの各ペアとそれらに囲まれた任意のテキストを強調表示します。

手動または自動でペイロードマーカーを配置できます。Burpの他の場所からリクエストを送信した場合、Burpはペイロードが配置されそうな箇所を推測し、それに応じてペイロードマーカーをセットします。リクエストテンプレートエディターの横にあるボタンを使用して、デフォルトのペイロードマーカーを変更できます。

  • §追加 - テキストが選択されていない場合、カーソル位置に１つのペイロードマーカーが挿入されます。テキストが選択されている場合は、選択したテキストを囲むペアのマーカーが挿入されます。
  • §クリア- テンプレート全体またはテンプレートの選択した部分から、すべてのマーカーを削除します。
  • 自動§- ペイロードに応じてペイロードマーカを置く適切な位置を推測します。ファジングに適した場所を素早くマークするのに便利ですが、攻撃をカスタマイズするには手動での調整が必要になります。テキストを選択した場合、選択したテキスト内のみにマーカーが配置されます。それ以外の場合、リクエストテンプレート全体の中で配置されます。マーカーの自動配置は、URLクエリーストリングパラメーター、ボディパラメーター、cookies、マルチパートパラメーター属性 (例えばファイルアップロードのファイル名)、XMLデータと要素の属性値、JSONパラメーターなど、様々なリクエストパラメーターのペイロードに配置されます。Intruderメニューのオプションで、自動ペイロードが既存のパラメーター値を置換するか追記するか、設定ができます。メッセージボディ全体ではなく、リクエストの一部にXMLやJSONで構造化されたデータが含まれている場合、フォーマットされたデータブロックを手動で選択し"自動"ボタンを押すと、その範囲内に自動でペイロードを配置ができます。これは、マルチパートパラメーター値に XML または JSON 形式でデータが含まれている場合、便利です。
  • リフレッシュ - 必要であれば、リクエストテンプレートエディターの構文の色分けを更新します。
  • クリア - リクエストテンプレート全体を削除します。

注: Burp Scannerによるスキャンの挿入ポイントをカスタムするためにも、IntruderのペイロードポジションUIが使えます。これを行うには、Intruder内で通常の方法でリクエストテンプレートとペイロードのマーカーを設定し、Intruderメニューから、"定義した挿入ポイントを診断"を選択します。

攻撃タイプ

Burp Intruderは、様々な攻撃タイプをサポートしています - これらはペイロードポジションにペイロードを割り当てる方法を決定します。リクエストテンプレートエディター上のドロップダウンを使用して、攻撃タイプを選択できます。次の攻撃タイプがあります。

  • スナイパー(Sniper) - 単一のペイロードセットを使用します。各ペイロードポジションを順番に対象にし、そのポジションに各ペイロードを順番に配置します。リクエストで対象となっていないポジションは影響を受けません - ポジションマーカーが削除され、テンプレート上のポジションマーカに囲まれていたテキストは変更されません。この攻撃タイプは、多くのリクエストパラメーターに対する一般的な脆弱性のファジングに便利です。攻撃で生成されるリクエストの合計数は、ペイロードポジション数とペイロードセット内のペイロード数の積です。
  • 破城槌(Battering ram) - 単一のペイロードセットを使用します。ペイロードを列挙し、定義されたペイロードポジションのすべてに、一斉に同じペイロードを配置します。この攻撃タイプは、リクエスト内の複数の箇所(例えば、cookieと body パラメーター内にあるユーザー名)に同じ値を入れる必要がある攻撃に便利です。攻撃で生成されるリクエストの合計数は、ペイロードセット内のペイロード数です。
  • ピッチフォーク(Pitchfork) - 複数のペイロードセットを使います。それぞれのポジション(最大で20個)に対して異なるペイロードを使います。すべてのペイロードセットを同時に列挙し、定義されたそれぞれのポジションを１つのペイロードに置き換えます。言い換えると、最初のリクエストでは、ペイロードセット１の１番目のペイロードを１番目のペイロードポジションに置き、ペイロードセット２の１番目のペイロードを２番目のペイロードポジションに置きます。２つめのリクエストでは、ペイロードセット１の２番目のペイロードを１番目のペイロードポジションに置き、ペイロードセット２の２番目のペイロードを２番目のペイロードポジションに置きます。この攻撃タイプは、リクエスト内の複数箇所に関連はあるが異なる値(例えば、あるパラメーターはユーザー名、別のパラメーターではユーザー名に対応する既知のID番号)の入力を必要とする場合に便利です。攻撃で生成されるリクエストの合計数は、最小のペイロードセットのペイロード数です。
  • クラスター爆弾(Cluster bomb) - これは複数のペイロードセットを使います。それぞれのポジション(最大で20個)に対して異なるペイロードを使います。攻撃は、ペイロードのすべての組み合わせをテストするように、各ペイロードを反復します。すなわち、2 つのペイロードポジションが存在する場合、ペイロードセット ２ の最初のペイロードをポジション２に置き、ポジション１にペイロードセット１の全てのペイロードを列挙します; 次にペイロードセット２の２番目のペイロードをポジション２き置き、ポジション１にペイロードセット１の全てのペイロードを列挙します。この攻撃タイプは、リクエスト内の複数箇所で、関連性がないもしくは未知の異なる値(例えば認証試行で、あるパラメーターがユーザー名で別のパラメーターがパスワード)の入力を必要とする場合に便利です。攻撃で生成されるリクエストの合計数は、全ての定義されたペイロードセットのペイロード数の積です - これは非常に大きくなる可能性があります。
