ペイロードポジション

このタブでは、攻撃の対象、リクエストテンプレート、ペイロードマーカ、攻撃の種類(どの位置にペイロードを割り当てるかを決める方法)を設定します。

対象フィールド

対象フィールドは、Intruderの攻撃が送信されるプロトコル(HTTPまたはHTTPS)、ホスト(対象サーバのIPアドレスまたはホスト名)、ポート(HTTP/Sサービスのポート番号)を制御します。対象フィールドにもペイロードマーカを配置できますが、Burpは自動的にペイロードマーカを配置しないので注意してください。

対象に合わせてHostヘッダを更新チェックボックスは、対象フィールドを変更した場合に、リクエストテンプレートのHostヘッダをBurpが自動的に更新するかどうかを制御します。このオプションを無効にすると、任意のHostヘッダを特定の対象に送信できます。これは、HTTP Hostヘッダ攻撃を行う場合など、さまざまな場面で役立ちます。

リクエストテンプレート

メインのリクエストエディタを使用して、すべての攻撃の派生元となるリクエストテンプレートを定義します。Burpは攻撃リクエストごとに、リクエストテンプレートから、ペイロードマーカによって定義されている位置に1つまたは複数のペイロードを当てはめます。

対象フィールドとリクエストテンプレートの設定

対象フィールドとリクエストテンプレートを設定する最も簡単な方法は、攻撃したいリクエストをBurp内のどこかで選び、コンテキストメニューのIntruderに送信オプションを選択します。すると選択したリクエストがIntruderの新しいタブに送信され、対象とポジションタブが自動的に現れます。

ペイロードマーカ

ペイロードマーカは、§文字を使用して次のように機能します:

  • マーカの各ペアは、1つのペイロードの位置を指定します。
  • 必要に応じて、リクエストテンプレート内の一部のテキストを、マーカのペアの間に囲めます。
  • ペイロードを当てはめる際、マーカと囲まれたテキスト両方が、ペイロードに置き換えられます。
  • ペイロードポジションに割り当てられたペイロードがないとき、マーカは削除されますが、囲われたテキストは変更されません。

設定しやすくするため、Intruderは自動的にペイロードマーカの各ペアとそれらに囲まれた任意のテキストを強調表示します。

Burpの他の場所からリクエストを送信した場合、Burpはペイロードが配置されそうな箇所を推測し、それに応じてペイロードマーカをセットします。なお、ペイロードマーカは対象フィールドには自動的に設定されません。リクエストテンプレートエディタの横にあるボタンを使用して、デフォルトのペイロードマーカを変更できます。

  • §追加 - テキストが選択されていない場合、カーソル位置に1つのペイロードマーカが挿入されます。テキストが選択されている場合は、選択したテキストを囲むペアのマーカが挿入されます。
  • §クリア- テンプレート全体またはテンプレートの選択した部分から、すべてのマーカを削除します。
  • 自動§- ペイロードに応じてペイロードマーカを置く適切な位置を推測します。ファジングに適した場所を素早くマークするのに便利ですが、攻撃をカスタマイズするには手動での調整が必要になります。テキストを選択した場合、選択したテキスト内のみにマーカが配置されます。それ以外の場合、リクエストテンプレート全体の中で配置されます。マーカの自動配置は、URLクエリーストリングパラメータ、ボディパラメータ、cookie、マルチパートパラメータ属性(たとえばファイルアップロードのファイル名)、XMLデータと要素の属性値、JSONパラメータなど、さまざまなリクエストパラメータのペイロードに配置されます。対象フィールドにマーカは追加されません。Intruderメニューのオプションで、自動ペイロードが既存のパラメータ値を置換するか追記するかを設定できます。メッセージボディ全体ではなく、リクエストの一部にXMLやJSONで構造化されたデータが含まれている場合、フォーマットされたデータブロックを手動で選択してから自動ボタンを押すと、その範囲内に自動でペイロードを配置できます。これは、マルチパートパラメータ値に XML または JSON 形式でデータが含まれている場合、便利です。
  • リフレッシュ - 必要であれば、リクエストテンプレートエディタの構文の色分けを更新します。
  • クリア - リクエストテンプレート全体を削除します。

注

Burp Scannerによるスキャンの挿入ポイントをカスタムするためにも、IntruderのペイロードポジションUIが使えます。これを行うには、Intruder内で通常の方法でリクエストテンプレートとペイロードのマーカを設定し、Intruderメニューから、定義した挿入ポイントを診断を選択します。
