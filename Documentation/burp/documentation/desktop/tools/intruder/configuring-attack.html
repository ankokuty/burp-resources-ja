Burp Intruder攻撃の設定

メインのIntruder UIでは、複数の攻撃をそれぞれ個別のタブで同時に設定できます。Intruderにリクエストを送信すると、それぞれ独自の番号付きタブで開かれます。各攻撃設定タブには、攻撃を設定する複数のサブタブがあります。各タブの詳細は、次のリンクを参照してください:

  • Target - 攻撃対象サーバの詳細を設定します。
  • Positions - ペイロードポジションを含む攻撃用のリクエストテンプレートと、攻撃タイプ(ペイロードポジションにペイロードを適用する方法)を設定します。
  • Payloads - 攻撃中にペイロードポジションに配置される、１つまたは複数のペイロードセットを設定します。
  • Options - この攻撃に影響を与える他の多くのオプションを設定します。

Intruderでの新たな攻撃を簡単に作成するには、他のBurpツール(Proxy履歴 やTargetサイトマップなど)で関連するベースリクエストを選択し、コンテキストメニューのオプションで"Intruderに送信"を使います。

新しい攻撃タブが作成され、自動的にTargetとポジションタブに、ベースリクエストに関連する詳細が設定されます。必要に応じて自動設定されたペイロードポジション、ペイロードサブテーブル、その他のオプションを変更できます。

ペイロードを配置するであろう箇所をBurpが自動的に推測した結果が表示されているはずです。デフォルトで、すべてのリクエストパラメータとcookieの値にペイロードが配置されます。ペイロードマーカの各ペアは１つのペイロードポジションを定義し、ベースリクエストのどこかのテキストを囲んでいる場合、ペイロードポジションが使われる際にそこがペイロードのコンテンツに置き換えられます。詳細については、ペイロードポジションのヘルプを参照してください。

リクエストエディタの横にあるボタンでペイロードマーカの"追加"や"削除"ができます。リクエスト内の新しい場所にペイロードマーカを追加したり、他のマーカを削除してみて、効果を確認してください。ペイロードマーカの動作を理解したら、"自動§"ボタンをクリックしてBurpのデフォルトのペイロードポジションに戻してください。

次に、"ペイロード"タブに移動します。定義されたペイロードポジションに配置されるペイロードを定義します。デフォルト設定(ペイロードに"単純リスト"を使う)のままにし、リストにいくつかテスト文字列を追加します。"新規アイテムを入力"ボックスに独自の文字列を入力し、"追加"ボタンをクリックします。あるいは、"Add from list"プルダウンメニューを使い、組み込みペイロード文字列から"Fuzzing - quick"の選択します [Pro version]

Burp Intruderには攻撃設定を管理する多くの機能があります。これらの機能はIntruderメニューから使用できます。

現在のタブの攻撃設定を保存して、後でそれを再読み込みできます。読み込みや保存の際に、ペイロードポジション設定を含めるか除外するか選択できます。

ペイロードポジションを含んだ設定は、特定の攻撃についての完全な設定を保存できます。ペイロードを除外した設定は、他のベースリクエストテンプレートやペイロードポジションで再利用できる、汎用的な攻撃設定を保存できます。 - 例えば、リクエストの特定タイプをファジングする好みの設定などです。

2つの既存のタブ間や新しいタブに、攻撃設定をコピーできます。上記同様、"ペイロードポジション"を含めるか除外するか選択できます。

新規攻撃タブを開いた際に、Intruderがどのように攻撃設定を扱うか制御できます。デフォルトの攻撃設定を使う、または最初もしくは最後に開いたタブからコピーするか選択できます。

後者のオプションを使用すると、一般的な攻撃設定(例えばファジング用)を作成し、Intruderに送信した各新規リクエストに自動的に適用できます。

攻撃タブ自身の管理も簡単です。以下が可能です:

  • タブヘッダをダブルクリックして名前変更
  • ドラッグして並び替え
  • 一番右の"..."タブをクリックして新しいタブを開く
  • タブヘッダの X ボタンをクリックしてタブを閉じる

攻撃の開始

攻撃がすべて設定できたら、Intruderメニューから"攻撃開始"を選択して攻撃を起動できます。

各攻撃は新規ウィンドウで実行され、そこには結果の詳細と、(設定されていれば)全リクエストとレスポンスがあります。結果の解析や、後で深掘りするアイテムの特定を補助するさまざまな機能があります。
