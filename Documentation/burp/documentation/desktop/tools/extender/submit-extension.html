BAppストアへの拡張の投稿

Burp Suiteの拡張を書いた場合、BAppストアで利用可能なBAppにするために、ぜひ私たちに投稿してください。投稿プロセスは次の通りです。

投稿する前に

拡張を投稿する前に、拡張が受け入れられる可能性が高いことを確認してください。ガイドラインを読み、私たちが考える良い拡張とは何か確認してください。重要ないくつかのポイントは次の通りです:

  • 拡張は、Burp Suiteや既存のBAppの機能と重複しないでください。

  • 拡張は、Burp Suiteネットワークを使用する必要があります。

  • 拡張は、応答性の維持やクリーンなアンロードなど、コーディングのベストプラクティスに従う必要があります。

  • 拡張は、オフライン作業や大規模プロジェクトなどのBurp Suiteのユースケースでうまく機能する必要があります。

BAppは、作成者のソースコードからフォークされます。現在のBAppのソースコードを確認したい場合は、私たちのリポジトリを見てください。

拡張の投稿

拡張を投稿するには、以下が必要です:

  • 関連するすべてのソースコードが保存されているGitHubリポジトリへのリンク。

  • 名前と、拡張の機能の適切な説明。

  • 拡張の動作の詳細、使用方法、拡張の使用に必要なセットアップ情報。

これらがすべてできたら、私たちに連絡してもらえれば、拡張をレビューします。

拡張のレビュー

拡張を入手したら、コンパイルして、BAppストアに適しているかレビューします。次のようにガイドラインに照らし合わせてレビューします:

  • 拡張が、既存の機能や利用可能な拡張機能と重複していないこと。

  • ガイドラインへの準拠を確認するためのコードレビュー。

  • 重要な機能の使用状況をチェックする自動レビュー。

  • ウイルス/マルウェアスキャン。

  • 拡張の手動テスト。

不明な点や改善の提案がある場合は、折り返し連絡します。拡張が確認できたら、BAppストアで使用するかどうかについて連絡します。

BAppの更新

BAppを更新するプロセスは次の通りです:

  1. あなたのリポジトリのPortSwiggerのフォークに対してプルリクエストを作成してください。フォークされたリポジトリからのプルリクエストのみを受け付けます。

  2. プルリクエストを作成したことを、メールで連絡してください。

  3. 変更を確認し、PortSwiggerのフォークにマージします。

  4. 拡張をテストし、更新されたバージョンをBAppストアに公開します。
