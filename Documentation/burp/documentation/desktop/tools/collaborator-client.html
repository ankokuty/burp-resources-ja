Burp Collaboratorクライアント

Burp Collaboratorクライアントは、手動テスト中にBurp Collaboratorを使用するツールです。手動テストで使用するペイロードを生成し、Collaboratorサーバをポーリングしそのペイロードによるあらゆるネットワーク通信を取得します。

Burp Collaboratorクライアントを実行するには、BurpメニューからBurp Collaboratorクライアントを選択してください。

次の機能があります:

  • 指定した個数のCollaboratorペイロードを生成し、クリップボードにコピーできます。これらを手動テストで、たとえばBurp IntruderやRepeaterで使用できます。
  • ペイロードにCollaboratorサーバのFQDNを含めるか、あるいは一意なインタラクションIDのみ生成するか選択できます。
  • 定期的または必要な時にCollaboratorサーバをポーリングし、そのペイロードによるあらゆるネットワーク通信の詳細を取得できます。

Burp Collaboratorクライアントを使用する場合、次の点に注意してください:

  • 各Collaboratorクライアントウィンドウは別のコンテキストで開き、ペイロードの生成とポーリングを実行します。別のクライアントウィンドウ同士でペイロードや通信をやりとりする方法はありません。したがって、クライアントウィンドウを閉じてしまうと、ペイロードによって生じた通信を取得する方法がなくなります。
  • 各Collaboratorクライアントウィンドウは、ウィンドウを開いた時点のCollaboratorサーバ設定に関連づけられています。Collaboratorサーバ設定を変更した場合(たとえば、他のプライベートCollaboratorサーバを使用するなど)、その設定を使用するには新たなクライアントウィンドウを開く必要があります。
  • ペイロードに関連する通信内容をポーリングするために、ペイロードで使用されているドメインに自分のマシンから接続できるか確認する必要があります。

注

WAFのブラックリストによる誤検出を減らすため、公開されているCollaboratorサーバに定期的に新しいドメイン名を追加しています。Burp Collaboratorクライアントはデフォルトで、現在のバージョンのBurp Suite Professionalがリリースされた時点で利用可能な最新の公開Collaboratorドメインを常に使用します。現時点では、burpcollaborator.netかoastify.comのどちらかになる予定です。

Burp Collaboratorが最大限の効果を発揮できるよう、Burp Collaboratorクライアントを実行しているマシンが、これらの両方のドメインにポート80と443でアクセスできるようにしてください。
