Burp Collaboratorクライアント

Burp Collaboratorクライアントは、手動テスト中にBurp Collaboratorを使用するためのツールです。手動テストで使用するペイロードを生成し、Collaboratorサーバをポーリングしそのペイロードによるあらゆるネットワーク通信を取得します。

Burp Collaboratorクライアントを実行するには、Burpメニューから"Burp Collaboratorクライアント"を選択してください。

次の機能があります:

  • 指定した個数のCollaboratorペイロードを生成し、クリップボードにコピーできます。これらを手動テストで、例えばBurp IntruderやRepeaterで使用できます。
  • ペイロードにCollaboratorサーバのFQDNを含めるか、あるいは一意なインタラクションIDのみ生成するか選択できます。
  • 定期的または必要な時にCollaboratorサーバをポーリングし、そのペイロードによるあらゆるネットワーク通信の詳細を取得できます。

Burp Collaboratorクライアントを使用する場合、次の点に注意してください:

  • 各Collaboratorクライアントウィンドウは別のコンテキストで開き、ペイロードの生成とポーリングを実行します。別のクライアントウィンドウ同士でペイロードや通信をやりとりする方法はありません。したがって、クライアントウィンドウを閉じてしまうと、ペイロードによって生じた通信を取得する方法がなくなります。
  • 各Collaboratorクライアントウィンドウは、ウィンドウを開いた時点のCollaboratorサーバ設定に関連づけられています。Collaboratorサーバ設定を変更した場合(例えば、他のプライベートCollaboratorサーバを使用するなど)、その設定を使用するには新たなクライアントウィンドウを開く必要があります。

Burp Collaboratorクライアントの使用

状況によって、レスポンスの内容や応答時間などいずれにも、アプリケーションの応答に顕著な影響が現れない場合があります。このような状況でも、データベースからテスターのサーバへの帯域外ネットワーク接続をさせることで、脆弱性を検出できる場合があります。Burp Scannerは、Burp Collaborator機能を使ってこの手法を行います。

別々のクライアントウィンドウやBurp Collaborator間で、ペイロードの共有や通信は行われません。したがって、クライアントウィンドウを閉じたり、スキャナによって生成されたペイロードを使用すると、そのペイロードから生じる通信を取得する方法はありません。

Burp Collaboratorクライアントを実行するには、Burpメニューから"Burp Collaboratorクライアント"を選択してください。
