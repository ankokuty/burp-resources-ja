Inspectorとメッセージエディタ設定

Inspectorとメッセージエディタ設定では、Inspectorパネルの表示位置の調整や、表示する情報の制御ができます。Inspector設定を開くをクリックすると、Inspector設定ダイアログが表示されます。

ダイアログには2つのタブがあります:

  • Inspector - Inspectorパネルに表示するウィジェットと、画面のどこにパネルを表示するかを定義する設定です。
  • メッセージエディタ - メッセージエディタのリクエストとレスポンスに、どのタブを表示するかを定義する設定です。

ウィジェット

ウィジェット設定では、Inspector内に表示されるウィジェットの表示方法を調整できます。次のことが可能です:

  • Inspectorパネルにどのウィジェットを表示するか選択できます。
  • 各ウィジェットをデフォルトで展開するかどうか選択できます。
  • ウィジェットのセル内でテキストを折り返すかどうか選択できます。
  • 上へと下へボタンを使って、ウィジェットのリストを並び替えられます。

位置と表示オプション

デフォルトのポジションと表示オプション設定では、Inspectorパネルのデフォルトレイアウトを設定できます。次のことが可能です:

  • パネルのデフォルトポジション(左または右)を設定できます。
  • パネルの表示モードを設定できます。Inspectorはデフォルトで自動展開に設定されており、利用可能な画面スペースに応じてパネルの展開/折りたたみをします。Inspectorを折りたたんだままにするには、常に折りたたみラジオボタンを選択します。

Inspector設定ダイアログは全体で共通で、Burp SuiteのすべてのツールでInspectorを使用する際に適用されます。

関連ページ

詳細は、Inspectorドキュメントを参照してください。

メッセージエディタ設定

メッセージエディタタブでは、次が可能です:

  • リクエストのメッセージエディタでツールバーに表示されるビューを編集できます。
  • レスポンスのメッセージエディタでツールバーに表示されるビューを編集できます。
  • 各ビューのテキストを折り返すかどうかを制御できます。
  • ビューのリストを並べ替えられます。

注

Proxy履歴のような読み取り専用のコンテキストでは、対応するコンテンツがメッセージに含まれている場合にのみ、選択したビューを表示します。例えば、リクエストにcookieが1つ以上含まれていない限り、Cookieタブは表示されません。

Burp Repeaterのような編集可能なコンテキストでは、すべてのビューを表示します。これにより、元のリクエストに存在しなかったアイテムでも追加できます。

関連ページ

詳細は、メッセージエディタドキュメントを参照してください。

Inspectorとメッセージエディタ設定は、ユーザ設定です。あなたのマシンにインストールされたすべてのBurpに適用されます。

HTTPメッセージ表示

HTTPメッセージエディタ内での、HTTPメッセージの表示方法を制御します。次の設定ができます:

  • フォント - 使用するフォントのスタイルとサイズです。Burpがフォントスムージングを使用するかどうかも指定できます。
  • リクエストの構文を色付けする - リクエストの構文が色付けされます。これには、JavaScript、JSON、CSSコンテンツが含まれます。
  • レスポンスの構文を色付けする - レスポンスの構文が色付けされます。これには、JavaScript、JSON、CSSコンテンツが含まれます。
  • デフォルトで整形表示 - Burpが自動的に、整形されていないデータ、マークアップ、コードに対して、正規化された改行やインデントを追加して整形して表示するかどうかを制御します。JSON、XML、HTML、CSS、JavaScriptの整形表示をサポートしています。

関連ページ

  • 整形表示の詳細は、テキストエディタを参照してください。
  • メッセージエディタの詳細は、Burp Suiteメッセージエディタを参照してください。

HTTPメッセージ表示設定は、ユーザ設定です。あなたのマシンにインストールされたすべてのBurpに適用されます。

文字セット

HTTPメッセージを生で表示する際に、さまざまな文字セットを処理する方法を制御します。使用可能な設定は次の通りです:

  • メッセージヘッダに基づいて各メッセージの文字セットを自動的に認識する。これがデフォルト設定で、さまざまな文字セットを使用するメッセージを同時に作業できます。
  • すべてのメッセージでシステムのデフォルト文字セットを使用する。
  • メッセージを(ASCIIエンコードを使用して)生バイト列で表示し、拡張文字セットの処理を行わない。
  • すべてのメッセージに指定した文字セットを使用する。リストから必要な文字セットを選択します。

HTTPヘッダは常に生形式で表示されます。文字セットエンコーディングの設定はメッセージボディにのみ適用されます。

注

一部の文字セットで必要な字形の中には、すべてのフォントでサポートされていないものがあります。拡張または特殊な文字セットを使用する場合は、まずCourier NewやDialogなどのシステムフォントを試してみてください。

文字セット設定は、ユーザ設定です。あなたのマシンにインストールされたすべてのBurpに適用されます。

HTMLレンダリング

HTTPメッセージエディタ内のRenderビューは、ブラウザに表示されるのとほぼ同じようにHTMLコンテンツを表示します。ページのレンダリングに必要な場合、追加のHTTPリクエストを送信する設定では、HTMLコンテンツを完全にレンダリングするために必要な追加のHTTPリクエスト(たとえば、埋め込み画像など)を送信するか制御します。

この設定を選択すると、BurpのHTMLレンダリングの品質が向上しますが、速度が若干犠牲になります。また、Burpが対象アプリケーションに行う全体的なリクエスト数も増加します。

関連ページ

詳細は、メッセージエディタドキュメントを参照してください。

HTMLレンダリング設定は、ユーザ設定です。あなたのマシンにインストールされたすべてのBurpに適用されます。

HTTPメッセージ検索

HTTPメッセージ検索設定は、メッセージエディタウィンドウの下部にある検索バーのデフォルトを制御します。使用可能な設定は次の通りです:

  • 大文字小文字を区別 - デフォルトで大文字小文字を区別して検索を行うかどうかを定義します。
  • 正規表現 - 検索語をデフォルトで正規表現として扱うか、リテラル文字列として扱うかを定義します。
  • テキスト変更時にマッチした箇所に自動スクロール - 新しいテキストが表示されたときに、最初に一致して強調表示されている箇所まで自動でスクロールするかどうかを指定します。これは、Proxy履歴のレスポンスから特定の表現を探す場合などに便利です。このオプションを選択して新しいアイテムを選択すると、最初の検索一致箇所に表示が自動でスクロールします。

HTTPメッセージ検索設定は、ユーザ設定です。あなたのマシンにインストールされたすべてのBurpに適用されます。
