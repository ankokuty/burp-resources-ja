ウェブサイトのスキャン

Burp Scannerは、Webサイトのコンテンツと脆弱性をスキャンする作業を自動化します。設定に応じてScannerは、アプリケーションをクロールしてコンテンツと機能を探索し、アプリケーションを診断して脆弱性を発見します。

  • Burp Scannerの仕組みについて詳細を参照する

スキャンの開始

スキャンは、さまざまな方法で開始できます:

  • 指定したURLからスキャン。1つ以上の指定したURL内のコンテンツをクロールし、必要に応じてクロールしたコンテンツを診断します。これを行うには、Burpダッシュボードで、"新規スキャン"ボタンをクリックしてください。スキャンランチャーが開き、スキャンの詳細を設定できます。
  • 選択したアイテムをスキャン。特定のHTTPリクエストに、診断のみ(クロールなし)を実行できます。これを行うには、Burp内の任意の場所で1つ以上のリクエストを選択し、コンテキストメニューから、"スキャン"メニューを選択してください。スキャンランチャーが開き、スキャンの詳細を設定できます。
  • ライブスキャン。ライブスキャンを使用すると、ProxyやRepeaterツールなど、他のBurpツールが処理したリクエストに自動的にスキャンができます。どのリクエストを処理するか、コンテンツの探索か脆弱性の診断をするかどうかを正確に設定できます。これを行うには、Burpダッシュボードで、"新規ライブタスク"ボタンをクリックしてください。ライブスキャンランチャーが開き、タスクの詳細が設定できます。

スキャンの設定

複数のスキャンを並行して起動でき、各スキャンには、スキャンの実行方法を正確に決定する個別の設定オプションがあります。設定には2つの重要な領域があります:

  • クロールオプション。これらのオプションは、最大リンク深度、スピードと網羅性によるクローラの最適化方法、クロール範囲の制限などの動作を制御します。続きを読む
  • 診断オプション。これらのオプションは、挿入ポイントの処理や、使用する検出手法などの動作を制御します。軽量で純粋なパッシブ解析から、重厚な侵入型スキャンまで、実行される診断アクティビティのタイプを制御する上でこれらのオプションが非常に重要です。続きを読む

スキャンアクティビティの監視

さまざまな方法でスキャンの進行状況と結果を監視できます。

  • Burpダッシュボードには各タスクの進捗状況が表示され、問題アクティビティログには全てのスキャンタスクによって報告された問題が表示されます。
  • 個々のスキャンのタスク詳細ウィンドウを開くと、そのスキャンのみの問題アクティビティログや、アプリケーションタスクの診断アイテムの詳細が表示できます。
  • Targetサイトマップには、発見された全てのコンテンツと問題が表示されます。

レポート

Burp Scannerが発見した問題のHTML形式のレポート生成ができます。他のツールへのインポートに適したXML形式でも、問題をエクスポートできます。
