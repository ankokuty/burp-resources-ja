Webサイトのスキャン

Burp Scannerは、Webサイトのコンテンツと脆弱性をスキャンする作業を自動化します。設定に応じてScannerは、アプリケーションをクロールしてコンテンツと機能を探索し、アプリケーションを診断して脆弱性を発見します。デフォルトでBurpの内蔵ブラウザを使用して、ブラウザを利用したスキャンによる最大限の網羅性を確保します。ユーザの認証情報セットを提供すると、認証されたユーザのみがアクセスできるコンテンツを検出して診断ができるようになります。完全なログインシーケンスをインポートすると、Burp Scannerはシングルサインオンを含むより複雑なログインメカニズムも処理できます。

  • Burp Scannerの仕組みについて詳細を参照する

スキャンの開始

スキャンは、さまざまな方法で開始できます:

  • 指定したURLからスキャン。1つ以上の指定したURL内のコンテンツをクロールし、必要に応じてクロールしたコンテンツを診断します。これを行うには、Burpダッシュボードで新規スキャンボタンをクリックしてください。スキャンランチャが開き、スキャンの詳細を設定できます。
  • 選択したアイテムをスキャン。特定のHTTPリクエストに、診断のみ(クロールなし)を実行できます。これを行うには、Burp内の任意の場所で1つ以上のリクエストを選択し、コンテキストメニューから、スキャンメニューを選択してください。スキャンランチャが開き、スキャンの詳細を設定できます。
  • ライブスキャン。ライブスキャンを使用すると、ProxyやRepeaterツールなど、他のBurpツールが処理したリクエストに自動的にスキャンできます。どのリクエストを処理するか、コンテンツの探索か脆弱性の診断をするかどうかを正確に設定できます。これを行うには、Burpダッシュボードで、新規ライブタスクボタンをクリックしてください。ライブスキャンランチャが開き、タスクの詳細が設定できます。
  • インスタントスキャン。コンテキストメニューから瞬時にアクティブまたはパッシブスキャンを起動もできます。つまり、スキャンランチャを開かなくても、脆弱性をすばやく確認できます。リクエストを右クリックすると、これらのオプションにアクセスできます。または、インスタントスキャンを起動するホットキーも設定できます。

スキャンの設定

複数のスキャンを並行して起動でき、各スキャンには、スキャンの実行方法を正確に決定する個別の設定オプションがあります。設定には2つの重要なエリアがあります:

  • クロールオプション。これらのオプションは、最大リンク深度、スピードと網羅性によるクローラの最適化方法、クロール範囲の制限などの動作を制御します。また、ブラウザによるスキャンやAPIスキャンなど、Burp Scannerのさまざまな機能の一部を有効または無効化もできます。
  • 診断オプション。これらのオプションは、挿入ポイントの処理や、使用する検出手法などの動作を制御します。これらのオプションは、軽量の完全に受動的なパッシブ分析から重量級の侵入的スキャンまで、実行される診断アクティビティのタイプを制御する上で非常に重要です。

独自のカスタム設定の作成も、スキャンランチャーのスキャン設定セクションにあるライブラリから選択ボタンを使って設定ライブラリから組み込み設定の読み込みもできます。

設定が作成されていない、または読み込まれていない場合は、Burp Scannerのデフォルト設定が使用されます。Burp Scannerのデフォルト設定は、パフォーマンスと網羅性のバランスが取れており、ほとんどのWebアプリケーションに適しています。

注

Burp Suiteの設定ライブラリには組み込み設定が用意されているので、これらのオプションを素早く設定できます。

スキャンアクティビティの監視

さまざまな方法でスキャンの進行状況と結果を監視できます。

  • Burpダッシュボードには各タスクの進捗状況が表示され、問題アクティビティログにはすべてのスキャンタスクによって報告された問題が表示されます。
  • 個々のスキャンのタスク詳細ウィンドウを開くと、そのスキャンのみの問題アクティビティログや、アプリケーションタスクの診断アイテムの詳細が表示できます。
  • Targetサイトマップには、発見されたすべてのコンテンツと問題が表示されます。

レポート

Burp Scannerが発見した問題のHTML形式のレポート生成ができます。他のツールへのインポートに適したXML形式でも、問題をエクスポートできます。
