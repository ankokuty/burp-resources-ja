モバイルデバイスのトラブルシューティング

このセクションでは、Burp Suiteを使用してモバイルアプリをテストしようとする際の、一般的な問題を解決する簡単な手順をいくつか紹介します。

BurpのCA証明書をインストールしたのに、iOSでHTTPS URLにアクセスできない

Burpで、ダッシュボードタブに移動し、イベントログを見てください。接続リセットやリモートホストがハンドシェイクを終了しましたなどのエラーが大量に見つかる場合は、TLS 1.3との互換性が原因である可能性があります。

Proxyリスナー設定で、ProxyがTLS1.3を使用しないよう無効に設定できます。

  1. Burpで、Proxy > オプションタブに移動します。

  2. モバイルデバイスに使用するProxyリスナーを選択し、編集をクリックします。

  3. Proxyリスナーを編集ダイアログで、TLSプロトコルタブに移動します。

  4. カスタムプロトコルを使用するを選択し、リストからTLSv1.3の選択を解除します。

これが問題だった場合は、通常どおりHTTPS URLにアクセスできるようになり、イベントログに表示されるエラーが少なくなります。それでも問題が解決しない場合は、サポートチームにお問い合わせください。
