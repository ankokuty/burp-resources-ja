プロジェクトの選択

一度のみ実行する必要があるインストールとライセンスのアクティベーションが完了すれば、次回からはBurpを起動するとスタートアップウィザードが直接表示されます。ウィザードの最初のステップは、使用するBurpプロジェクトの選択または作成です。

次のオプションから選択できます:

  • 一時プロジェクト - 保存する必要がない素早いタスクに適しています。すべてのデータはメモリ内に保持され、Burpを閉じると失われます。
  • 新規ディスクプロジェクト - 新しいプロジェクトが作成され、データをBurpプロジェクトファイルに保存します。このファイルには、プロジェクトの全データと設定が保存され、作業するにつれデータが追加されていきます。プロジェクト名の指定も可能です。プロジェクト名は後でBurpメニューから変更できます。
  • 既存のプロジェクトを開く - Burpプロジェクトファイルから既存のプロジェクトを再開します。最近開いたプロジェクトのリストが表示され、簡単に選択できます。デフォルトでは、既存のプロジェクトを開くと、進行中の自動化されたタスクは最初は一時停止されます。これは、以前設定されていた対象に意図せずリクエストを送信しないためです。必要なら、このオプションを解除できます。

Burpプロジェクトでの作業

別のBurpインストールからのプロジェクトを開始

別のBurpインストールで作成された既存のプロジェクトを開いた場合、プロジェクトの完全な所有権を取得するかどうかプロンプトが表示されます。

プロジェクトにはBurp Collaboratorが関連付けられており、その通信を検索するための識別子がプロジェクトファイル内に保存されているため、この選択が必要です。２つのBurpインスタンスで同じ識別子を共有し作業すると、Collaboratorベースの診断項目を見逃す、または正しく報告されない可能性があります。他のBurpインスタンスがそのプロジェクトで作業していない場合のみ、他のBurpインストールからプロジェクトの完全な所有権を取得してください。
