設定

設定ライブラリを使用して、個々のタスクに応じて、異なるBurp設定を利用できます。例えば、スキャンの種類ごとに、異なる設定が作成できます。または、作業を行うクライアントごとに固有の設定を読み込む必要があるかもしれません。設定ファイルの形式で、保存や読み込みもできます。

設定ライブラリ

Burpの設定ライブラリには、様々なタスクタイプの設定オプションが格納されており、スキャンの開始のようなアクションを実行する際に簡単に利用できます。

設定ライブラリには次のものがあります:

  • 組み込み設定 - 一般的な用途で便利な、事前定義された設定です。例えば、高速クロールの実行や、重大な脆弱性のみの診断、などです。
  • カスタム設定 - ユーザーが定義し管理する設定です。

Burpメニューや、スキャンランチャーなど他の関連機能から、Burpの設定ライブラリにアクセスできます。

設定ライブラリの各アイテムは、"クロール"など特定の機能に適用されます。新しいカスタム設定の作成や、既存のものを編集する際は、選択した機能の設定エディタが表示されます。通常、指定された機能で定義できる複数の設定範囲があります。設定エディタでは、その設定で定義する範囲を選択できます。設定を定義していない範囲には、デフォルトまたは既存の値が使用されます。多くのBurp機能で複数の設定を選択できるのでこの挙動が便利です; これらは順番に適用され、複数の設定から最終的な設定を構築できます。

各設定には個別の名前を付ける必要があります。設定ファイルとして、設定のエクスポートまたはインポートができます。設定の詳細は、JSON形式で記述されます。

ユーザーとプロジェクトの設定ファイル

ユーザーレベルとプロジェクトレベルのオプション管理に、個別の設定ファイルが使用されます。

ユーザー設定ファイルには、個々のユーザー環境とUIに関する設定が書かれており、内容は次の通りです:

  • ユーザーオプションタブの全て
  • 設定された拡張を含む、Extenderツール
  • ターゲットサイトマップで選択したビューなど、他のツールのUI関連の設定

プロジェクト設定ファイルには、個々の対象アプリケーションで実行される作業に関する設定が書かれており、内容は次の通りです:

  • プロジェクトオプションタブの全て
  • ProxyやScannerなど、個々のBurpツールの非UI関連の設定

設定ファイルの読み込みと保存

様々な方法で設定ファイルの読み込みと保存ができます:

  • Burpの設定ライブラリで、設定ファイルとして、設定のエクスポートまたはインポートができます。
  • Burpメニューから、ユーザーレベルおよびプロジェクトレベルの全ての設定を、読み込みや保存ができます。
  • Burpの至る所にある個々の設定パネルで、"オプション"ボタンでそのパネルの設定を、読み込みや保存ができます。
  • 開始ウィザードで、プロジェクトの作成または再開をするとき、プロジェクトレベルの設定を読み込む設定ファイルを指定できます。
  • コマンドラインからBurpを実行するとき、コマンドライン引数を使用して、１つまたは複数の設定ファイルを指定してユーザーレベル及びプロジェクトレベルの設定を読み込めます。
  • Burp拡張からはAPI経由で設定ファイルの読み込みと保存ができます。

設定ファイルのフォーマット

設定ファイルはJSON形式を使用しています。JSON内で使用される構造と名前スキームは、Burp UIでの設定表示方法に対応します。個別の目的に応じた設定ファイル作成するには、Burp UIで必要な設定を作成し、そこから設定ファイルの保存をする方法が、最も簡単です。設定ファイルの内容は人間が判読可能でそれ自身がドキュメントにもなっているので、必要に応じて既存の設定ファイルを手動で編集できます。
