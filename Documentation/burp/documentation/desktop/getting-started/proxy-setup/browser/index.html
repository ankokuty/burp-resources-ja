Burpと連携するよう外部ブラウザを設定する

注: Burpを使用した手動テストに外部ブラウザを使用する場合にのみこれらの手順が必要です。必要に応じてBurpの内蔵ブラウザが使用でき、これは既にBurp Proxyで動作するように事前設定されています。内蔵ブラウザにアクセスするには、"プロキシ" > "インターセプト"タブに移動し、"ブラウザを開く"をクリックします。

プロキシリスナーの稼働を確認したら、それがHTTPプロキシサーバとして使用されるようにブラウザを設定する必要があります。それにはブラウザのプロキシ設定で、HTTPとHTTPS両方のプロトコルでアドレス(デフォルトは127.0.0.1)とポート(デフォルトは8080)を変更し、例外設定を空にします。これにより、すべてのHTTPおよびHTTPSトラフィックがBurpを通過するようになります。詳細な設定方法は、ブラウザとバージョンによって異なります。Burpで使用するブラウザに応じて、次の関連セクションを参照してください。

  • Burpと連携するようFirefoxを設定する
  • Burpと連携するようChromeを設定する
  • Burpと連携するようSafariを設定する
  • Burpと連携するようInternet Explorerを設定する

ブラウザのプロキシ設定の確認

ブラウザの設定ができたら、正しく動作しているかテストするために次の手順を実行する必要があります。次に説明する方法で何も起こらない場合は、ブラウザの設定に問題があります。この場合、トラブルシューティングページを参照してください。

  1. プロキシリスナーがアクティブであるか確認し、選択したブラウザが設定済みか確認します。
  2. Burpを実行している状態で設定したブラウザを開き、ブラウザでどこかHTTPのURL(まだHTTPSは使わないでください)にアクセスします。ブラウザがリクエストの完了を待っている状態、つまりページを読み込もうとしてスタックしているように見えるはずです。これは、ブラウザが送信しようとしているHTTPリクエストをBurpがインターセプトしたためです。
  3. Burpで、"Proxy"タブに移動し、"インターセプト"サブタブを開きます。これらのタブが両方とも強調表示されるはずです。"インターセプト"タブで、メインパネルにインターセプトされたHTTPリクエストが表示されます。
  4. "インターセプト有効"というボタンに注目してください。クリックすると"インターセプト無効"に変わり、Burpからリクエストが放出されます。
  5. ブラウザに戻ります。これで、通常のブラウジング時と同じように、要求されたページが読み込まれ表示されます。

上記のすべてができた場合、Burp Suiteで外部ブラウザを使用するための必須設定が完了しています。ただし現時点では、HTTPだけを使用するWebアプリケーションのみをテストできます。外部ブラウザを使用してHTTPS URLにアクセスしようとすると、接続がブロックされるでしょう。したがって、HTTPSを使用するアプリケーションをテストできるように、BurpのCA証明書をインストールする最後の追加手順を強く推奨します。
