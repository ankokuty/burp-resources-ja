モバイルテスト

Burp Suiteを使用して、モバイルアプリケーションのセキュリティテストが実行できます。これを行うには、モバイルデバイスのトラフィックをBurp Proxyを経由するように設定するだけです。すると、モバイルアプリで処理されるHTTP/Sのリクエストやレスポンスをインターセプト・表示・変更でき、通常の方法でBurpを使用したペネトレーションテストが実行できます。

プロキシ設定に問題がある場合やSSL証明書のピニングがされている場合、モバイルアプリケーションからのHTTP/Sトラフィックのインターセプトは容易ではないかもしれません。

Burp Suiteモバイルアシスタントは、Burp SuiteによるiOSアプリのテストを容易にするためのツールです。次の主要機能をサポートしています:

  • iOSデバイスのシステム全体のプロキシ設定を変更し、実行中のBurpインスタンスにHTTP(S)トラフィックを容易にリダイレクトできます。
  • 選択されたアプリのSSL証明書ピニングの回避し、Burp SuiteがHTTPS接続を解読しすべてのトラフィックをインターセプト、調査、修正できるよう試みます。

Burp Suiteモバイルアシスタントは現在、iOSバージョン8.0以降で動作するデバイスをサポートしています。

詳細は、次のリンクを参照してください:

  • モバイルアシスタントをインストール
  • モバイルアシスタントの使い方

注: Burp Suiteモバイルアシスタントは、データの可用性・機密性・完全性が必要な状況では使用しないでください。モバイルアシスタントは、通信のセキュリティを大幅に低下させる方法で、挿入されたアプリを変更します。
