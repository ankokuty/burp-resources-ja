BAppストアへの拡張の投稿

Burp Suite用の拡張を開発した場合、BAppとして投稿できます。拡張が受入基準を満たしていれば、BAppストアから利用できるようにします。

ステップ1 - 受入基準を確認する

拡張が受入基準を満たしていて、受け入れられる可能性が高いかどうかを確認してください。

ステップ2 - 拡張を投稿する

拡張を投稿するには、次が必要です:

  • 関連するすべてのソースコードが保存されているGitHubリポジトリへのリンク。
  • 名前と、拡張の機能の適切な説明。
  • 拡張の動作の詳細、使用方法、拡張の使用に必要なセットアップ情報。

これらがすべて揃ったら、レビューを依頼するために私たちに連絡してください。

ステップ3 - 拡張のレビューを受ける

私たちは、拡張をコンパイルし、BAppストアに適しているかどうかを確認します:

  • 拡張が、既存の機能や利用可能な拡張と重複していないか確認します。
  • ガイドラインへの準拠を確認するためにコードをレビューします。
  • 重要な機能の使用状況を確認するために自動レビューをします。
  • 拡張が、ウイルス対策ソフトやマルウェアを起動させるかどうか、スキャンを実行します。たとえば、暗号化されていないペイロードが誤検出されることがあります。
  • 拡張を手動でテストします。

これらのプロセスを通じて、質問や改善提案など連絡を取り合うことを心がけています。

レビュー終了後、拡張をBAppストアに掲載するかどうかをお知らせします。

BAppの更新

BAppを更新するには:

  1. あなたのリポジトリからフォークした、PortSwiggerのリポジトリに対してプルリクエストを作成してください。プルリクエストはフォークしたリポジトリからのみ受け付けています。
  2. プルリクエストのリンクをメールで送信してください。
  3. 私たちはその変更点をレビューし、PortSwiggerのフォークにマージします。
  4. 拡張をテストした後、BApp Storeで更新したバージョンを公開し、公開したらお知らせします。
