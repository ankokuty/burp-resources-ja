拡張によるシステム影響の監視

拡張を利用すると、Burp Suiteでさまざまなタスクを実行できます。それらのタスクが大量のリソースを使用する場合や時間がかかる場合に、Burp Suiteのパフォーマンスに影響を与える可能性があります。

特定の拡張がシステムパフォーマンスに与える影響を見積もるには:

  1. 拡張 > BAppストアに移動し、BAppストアを表示します。
  2. 拡張を選択し右側のパネルをスクロールして、システム影響の概算を表示します。

  3. 各カテゴリの影響を確認します:

    • メモリ - BAppがBurp Suiteのメモリ使用量に与える影響。
    • CPU - BAppがCPUにかける負荷。
    • 時間 - Burp Suiteの速度に与える全体的な影響。ここには、インターフェースの応答性や、ツールが作業を完了するまでにかかる時間などが含まれます。
    • Scanner - スキャンが完了するまでの時間に影響を与える可能性。
    • 総合 - これらすべてのカテゴリーで最も高い影響。

注

システム影響の評価は目安で、BAppによるパフォーマンスへの影響を完全に反映していない可能性があります。カスタムタブやコンテキストメニューオプションを追加するBAppは、十分なテストができません。

拡張による全体的な影響

複数の拡張を同時に使用すると、性能への影響が累積します。この概算は、全体的なシステム影響を参照してください。BAppストアの上のパネルにあります。

注

さまざまなPython拡張やRuby拡張を読み込んだり、拡張のロードとアンロードを繰り返すと、メモリ問題が発生する場合があります。次のようなエラーが表示される場合があります:

java.lang.OutOfMemoryError: PermGen space

この問題を回避するには、Burpを起動するときのコマンドラインオプションに-XX:MaxPermSizeを追加し、より多くの PermGenストレージが割り当てられるようJavaを設定します。例:

java -XX:MaxPermSize=1G -jar burp.jar
