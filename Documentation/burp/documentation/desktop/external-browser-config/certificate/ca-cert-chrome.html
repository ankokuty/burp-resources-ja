ChromeへBurpのCA証明書をインストール

BurpのCA証明書をインストールする前に、Proxyリスナーが動作しており、Burpが使われるようブラウザの設定ができているか確認します。

残念ながら、ChromeにBurpのCA証明書をインストールする手順は、オペレーティングシステムごとに若干異なります。

ChromeへBurpのCA証明書をインストール - WindowsとMacOS

他のブラウザとは異なり、Chromeには独自のトラストストアがありません。これは、BurpのCA証明書を直接Chromeにインストールしないということです。代わりに、コンピューターの内蔵ブラウザ(Windowsの場合はInternet Explorer、MacOSの場合はSafari)にBurpのCA証明書をインストールすると、Chromeも自動的に証明書を使用します。

ChromeでBurpを使用する場合は、オペレーティングシステムの該当するブラウザにCA証明書をインストールする手順に従ってください:

  • MacOS (Safari)
  • Windows (Internet Explorer)

BurpのCA証明書をコンピュータの内蔵ブラウザに正常にインストールしたら、Chromeを再起動します。Burpを実行したまま、HTTPSのURLにアクセスしてみてください。すべて正常に動作していれば、セキュリティ警告なしでページを参照できるはずです。

ChromeへBurpのCA証明書をインストール - Linux

WindowsとMacOSでは、Chromeブラウザはホストコンピュータから証明書トラストストアを取得します。ただしLinuxでは、ブラウザの設定で証明書をインストールする必要があります。

Burpを実行している状態で、Chromeでhttp://burpsuiteにアクセスします。"Welcome to Burp Suite Professional"という比較的わかりやすいWebページが表示されるはずです。表示されない場合は、プロキシのトラブルシューティングページを参照してください。何が悪かったかによっては、自動的にそこに遷移するかもしれません。

ページの右上隅にあるCA Certificateをクリックして、独自のBurp CA証明書をダウンロードします。保存する場所をメモしておいてください。

ブラウザの右上隅にあるメニューを開き、設定をクリックして、Chromeブラウザの設定を開きます。

Chromeの設定で、証明書を検索し、セキュリティをクリックし、証明書の管理オプションを選択します。

証明書の管理ダイアログで、認証局タブに移動し、インポートボタンをクリックします。

参照をクリックし、先程ダウンロードしたcacert.derファイルを選択します。開くをクリックします。

ウェブサイトの識別でこの証明書を信頼しますオプションを選択します。

OKをクリックします。

org-PortSwiggerが認証局のリストに表示されるはずです。

Burpを実行したまま、HTTPSのURLにアクセスしてみてください。すべて正常に動作していれば、セキュリティ警告なしでページを参照できるはずです。
