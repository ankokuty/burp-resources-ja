手動テストシミュレータ

この機能にアクセスするには、Targetタブのサイトマップで、コンテキストメニューの"関連ツール"から"手動テストをシミュレート"を選択してください。

この機能は、あなたの生産性を全く向上させませんが、時々便利かも知れません。この機能は、不規則な間隔でランダムなURLやパラメータに一般的なテストペイロードを送信し、手動でテストをしているのと同様のトラフィックを生成します。サイトマップで選択したアイテムのみにリクエストが飛びます。

Burpはレスポンスについて何もしないので、この方法で何らかのバグを見つけることはないでしょう。あなたがちゃんと仕事をしているか誰かがWebアプリケーションのログをレビューするかもしれないと思うなら、長めの昼食・ジムでトレーニング・しびれるような呑み会、あるいはその他の気分転換をしている間、この機能が使えるでしょう。

"シミュレーション実行"チェックボックスで、手動テストシミュレータが起動または停止します。

さて、シミュレータがあなたの標準的な日給換算でいくら稼いだか示すカウンタはない方がいいですよね？隠し機能？誰か見つけた？
