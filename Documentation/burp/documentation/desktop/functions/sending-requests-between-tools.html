異なるツール間のリクエスト送信

異なるツール間でのリクエスト送信は、Burpのテストワークフローの重要な部分です。例えば、Burp Proxyで興味深いリクエストに注目し、それをBurp Intruderに送ってファジングができます。ファジングの結果に基づいて、同じリクエストをBurp Repeaterに送信し、特定の入力に対してより対象を絞った手動テストができます。

他のツールがBurpのどこに表示されていても、そのツールにリクエストを送信できます。これは、次の通りです:

  • Proxy > HTTP履歴やLoggerタブに表示されるリクエスト。
  • Proxy > インターセプト機能でインターセプトしたリクエスト。
  • Burpの任意の場所にあるメッセージエディタに表示されたリクエスト。

リクエストを他のツールに送るには、右クリックし、該当する[...]に送信オプションを選択します。

同じツールへのリクエスト送信

場合によっては、現在表示している同じツールへリクエストを送信できます。例えば、既にBurp RepeaterにあるリクエストでRepeaterに送信を選択すると、同じリクエストの別のインスタンスを含んだ新しいRepeaterタブが作成されます。

これはテストの整理に便利です。例えば、興味深い挙動をみつけて調査したいと思ったら、新しいタブを作成し、後で戻ってこられるようにしておきます。
