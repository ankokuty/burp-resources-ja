メッセージインスペクタ

メッセージインスペクタは、Burp Suite全体でメッセージエディタの右側に表示される折りたたみ可能なパネルです。HTTPやWebSocketメッセージの興味深い機能について、タブを切り替えずに素早く分析できます。インスペクタパネルとパネル内の列の両方を水平方向にサイズ変更できます。インスペクタ内で一度に複数アイテムの選択もできます。

HTTPメッセージの場合、インスペクタパネルには、表示されたメッセージの中にある潜在的に興味深い項目のリストが、最初に表示されます。これらは、次のカテゴリに分類されます:

  • クエリパラメータ
  • HTTPボディパラメータ
  • リクエストcookie
  • リクエストヘッダ
  • レスポンスヘッダ

各カテゴリの横の数字は、このタイプの項目が何個あったかを示しています。各カテゴリを展開すると、それらすべてのアイテムとそれぞれの値が表示されます。パラメータとcookieに表示される値は、自動的にURLデコードされます。個々のアイテムをドリルダウンして、それらに対してさまざまな一般的な操作が実行できます。

Burp Repeaterなどの編集可能なメッセージの場合、各カテゴリの下部にあるボタンを使用して、メッセージ内のアイテムを追加・削除・並び替えができます。各アイテムの値を編集し、変更結果をメインエディタに反映もできます。

エンコードされたデータをインスペクタで操作する

Burp Decoderと値をやり取りしたり、選択したテキストをコンテキストメニューから手動でデコードする代わりに、一部の同じ機能はインスペクタでもアクセスできます。

自動デコード

インスペクタに表示される初期値は、ほとんどの場合、すでにURLデコードされています。ただし、さらに解除する必要のあるエンコーディングレイヤが追加であるかもしれません。たとえば、クエリパラメータがBase64とURLエンコードの両方がされている場合があります。エンコードされたアイテムをドリルダウンすると、一連の適切な変換が自動的に実行され、その値を完全にデコードします。これにより、エンコードされたデータの操作がはるかに迅速かつ容易になります。現在インスペクタは、HTML・URL・Base64エンコーディングを処理できます。

各デコードステップとその結果の値が別々に表示されるので、舞台裏で何が起こっているかが正確に分かります。また、各ステップで使用されるデコードを手動で調整し、必要に応じて追加のデコードステップも追加できます。

選択した文字のデコード

ヘッダ・パラメータ・cookieの値をデコードするだけでなく、インスペクタを使用すると任意の文字シーケンスのデコードもできます。

編集可能なメッセージ上で2文字以上の文字を直接選択すると、インスペクタに"選択"ウィジェットが表示されます。ここに選択したテキストが表示され、アイテムをドリルダウンしたときに使用できるのと同様にスマートおよび手動のデコードができます。編集不可能なメッセージの場合、サポートされているエンコーディングが含まれている値を選択した場合にのみ、選択ウィジェットが表示されます。

エンコードされたデータの編集

インスペクタを使用すると、メッセージ内にあるエンコードされたデータの編集と操作がはるかに簡単になります。

インスペクタでアイテムをドリルダウンしていくと、デコードされた値が変更ができるよう、値の最後にカーソルが置かれます。入力すると、表示された一連のエンコーディングが行われ、変更が自動的に適用されます。すべてのエンコーディングが再適用された結果の値が、パネルの上部に表示されます。

"変更を適用"をクリックするかEnterキーを押すと、完全にエンコードされた新しい値でメッセージ内の値を上書きできます。適切なデコード手順のシーケンスが最初に使用されていれば、これは元の値と全く同じ方法でエンコードされるはずです。

この機能は、メッセージエディタでエンコードを直接手動で選択した値でも使用できます。

個々の文字をインスペクタで操作

メッセージエディタで個々の文字を直接選択すると、"選択"ウィジェットがインスペクタに表示されます。ただし、複数の文字を選択したときに表示されるデコード機能は使えません。その代わりに、ドロップダウンメニューを使用して、文字の16進または10進のコードポイントを表示できます。

編集可能なメッセージの場合、このコードポイントを編集し、変更を適用すると、メッセージ内で選択した文字を上書きできます。

印字不可文字の挿入

メッセージエディタ内の\nボタンをクリックすると、印字不可能文字を表示するかどうかを切り替えられます。さまざまな理由から、印字不可能文字を追加で挿入したい場合があるかもしれません。これをインスペクタで行うには、いくつかの方法があります:

  • 値を編集するときに、Shift + Enterを押すと、CRLF(\r\n)が挿入できます。
  • 編集可能なメッセージで個々の文字を選択すると、インスペクタで16進または10進のコードポイントを変更できるので、その文字を印字不可能文字に置換ができます。例えば、対象サーバでヌルバイト(\0)がどのように処理されるかを観察したい場合は、16進コードで00にすれば挿入できます。
