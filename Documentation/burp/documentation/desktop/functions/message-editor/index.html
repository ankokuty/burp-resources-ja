Burp Suiteメッセージエディタ

メッセージエディタは、HTTPリクエストとレスポンス、WebSocketメッセージの表示や編集をするためにBurp全体で使用されます。メッセージ自体を表示するだけではなく、メッセージを素早く分析し、Burpのコアワークフローを推進し、他の便利なタスクの実行をするための多くの機能があります。

メッセージエディタは主に次のパネルで構成されています:

  • テキストエディタ - メッセージ自体を表示します
  • 折りたたみ可能なInspectorパネル - HTTPメッセージやWebSocketの重要な詳細情報に素早くアクセスでき、タブを切り替えずにさまざまな基本的な操作が行えます。Inspectorパネルは、Burp Suiteの過去のバージョンにあったHex、Params、Headersタブを、効果的に置き換えます。

メッセージエディタの右上隅には、自分の好みに合わせてレイアウトを調整するためのアイコンが3つ並んでいます。次のオプションから選択できます:

  • 水平レイアウト: リクエストとレスポンスが左右に並べて配置されます。
  • 垂直レウアウト: リクエストとレスポンスが上下に並べて配置されます。
  • 結合ビュー: リクエストとレスポンスのどちらかがメッセージエディタペインに表示されます。対応するタブを使って交互に切り替えられます。

メッセージ分析ツールバー

各リクエストやレスポンスの上部には、さまざまなタイプのメッセージの分析に役立つ、さまざまな機能を提供するツールバーがあります。異なるビューを交互に切り替えると、メッセージの内容の表示方法を制御したり、さまざまな一般的な操作を実行したり、さらにメッセージを分析するために組み込まれたカスタム拡張が利用できます。

Rawビュー

このビューでは、テキストエディタはメッセージ全体を生の形式で表示します。テキストエディタには、構文分析、ホットキー、テキスト検索など便利な機能がいくつもあります。\nボタンを使用すると、印字不可能文字が表示されます。

Burp RepeaterなどBurpの一部のツールでは、リクエストを直接テキストエディタで編集もできます。

Actionメニューや、関連するメッセージのどこかをクリックすると、リクエストとレスポンス両方に対するツール全体で利用できるコンテキスト固有のアクションにアクセスできます。メッセージから1文字以上の文字を選択すると、その値をInspectorで操作できます。

Prettyビュー

このビューでは、Rawビューと同じ機能にすべてアクセスできます。主な違いは、テキストエディタの整形表示機能が有効になっていることです。これにより、HTTPメッセージ内のデータ、マークアップ、コードが、標準化されたインデントと改行で表示されるため可読性が大幅に向上します。

編集可能なメッセージで、かつサポートされているテキスト形式であれば、入力をするたびに可能な限り動的に整形しようとします。それ以外の場合、リクエストを送信するときにテキストが整形されます。

注

このビューは、サポートされているフォーマットのいずれかがメッセージのコンテンツに含まれている場合にのみ表示されます。

Hexビュー

メッセージの生の形式をHexエディタで表示します。1行に16バイトを配置し、各バイトの16進数値を表示します。Hexビューでメッセージを編集でき、文字か00からFFまでの2桁の16進数で指定して値を挿入できます。

選択したバイトはすべてInspectorに表示されます。Inspectorで直接か、テーブル内の値をダブルクリックすると、個々のバイトを直接編集できます。行番号をクリックするとバイトの行が選択され、選択した部分をInspectorで表示できます。

Hexビューは、次の場合に便利です:

  • 各文字のコードポイントを表示または編集する。
  • 印字不可能文字を表示または挿入する。
  • 一部のバイトや文字列を挿入または削除する。

このビューのコンテキストメニューには、さらに次の項目があります:

  • バイト挿入: 選択したバイトの前に新しく1バイト挿入します。
  • 複数バイト挿入 選択したバイトの前に、要求したバイト数を新しく挿入します。
  • 文字列挿入 選択したバイトの前に、指定した文字列を挿入します。
  • バイト削除 選択した1バイトを削除します。
  • 選択バイト削除 選択したバイト列を削除します。

Renderビュー

メッセージボディにHTMLや画像コンテンツを含むHTTPレスポンスに適用されます。ブラウザに表示されたときに表示される形式で、メッセージボディの内容をレンダリングしようとします。

拡張固有のビュー

一部のBurp拡張は、メッセージエディタにビューを追加します。これらのビューには、ツールバーの右側にあるドロップダウンメニューからアクセスできます。

拡張固有のビューをまだ開いたことがない場合は、メニューボタンにSelect extension...と表示されます。このボタンをクリックすると、現在ロードされているすべての拡張のリストが表示されます。拡張の1つを選択すると、カスタムビューに切り替わります。

拡張固有のビューを開くと、メニューボタンに拡張の名前が表示されます。対応するボタンをエディタでクリックすると、このビューとデフォルトビューを切り替えられます。ボタンの右側にある矢印を使用すると、表示する拡張を変更できます。

Actionsメニュー

Actionsメニューから、現在のリクエストで利用できるすべての範囲のコンテキスト固有のアクションに素早くアクセスできます。

メッセージエディタを使用する他の方法

メッセージエディタでは、次も行えます:

  • 通常のHTTPメッセージエディタビューで、印字不可能文字を四角で表示するかどうかを切り替えられるようになりました。これを行うには、メッセージエディタのツールバーの\nボタンを押します。
  • 文字のコードポイントを表示するには、HTTPメッセージで、印字可能、または印字不可能文字を選択します。Inspectorにエントリが表示され、文字の10進と16進コードポイントが表示されます。Burp Repeaterでのリクエストなど編集可能な場面では、この値を編集して選択した文字を上書きできます。
  • 既存のバイトのコードポイントをInspectorで編集すると、ヌルバイト、復帰、改行文字などの印字不可能文字を挿入できます。
  • カーソルをリクエストの適切な位置に置いてEnter/Returnキーを押すと、CRLFを挿入できます。
  • 1バイトを挿入するには、リクエスト内の目的の位置にカーソルを置き、任意のプレースホルダ文字を入力します。Inspectorを使用して、それを適切なコードポイントに編集します。
  • バイトを削除するには、リクエスト内の対応する文字を直接削除します。印刷不可能文字を削除するには、最初に\nボタンを使用して表示しておくことを推奨します。

注

Inspectorを使用して文字のコードポイントを編集する代わりに、選択範囲をURLエンコードし、16進コードをインラインで編集してから、選択範囲を元の形式にデコードする方が速い場合があります。これは、対応するホットキーを使用すると特に効果的です。

メッセージエディタのHTTP/2メッセージ

メッセージエディタは、HTTP/1構文を使用してHTTP/2メッセージを表現し、基本的に同等のHTTP/1リクエストがどのように見えるかを示します。変更を加えるたびに、Burpはバックグラウンドで自動的にHTTP/2に変換し、基になるHTTP/2リクエストを更新します。多くの場合、使用しているプロトコルによる影響はないため、一般的なテスト目的では、HTTP/2でも通常どおりメッセージエディタを使用できます。

このHTTP/1スタイルの表示では、HTTP/1構文の制限に縛られ、また有効なHTTP/2リクエストが生成されるように若干の正規化が必要です。HTTP/2特有のプロトコルレベルの問題をテストする場合には適していない場合があります。このような場合は、Inspectorを使用したHTTP/2の操作を推奨します。

詳細は、HTTP/2ドキュメントを参照してください。

コンテキスト固有のアクション

Actionsボタンをクリックすると、リクエストとレスポンスに対するコンテキスト固有の幅広いアクションにアクセスできます。あるいは、メッセージを右クリックして、コンテキストメニューからアクションも選択できます。使用可能なアクションは、メッセージタイプによって異なります。これらを以下に記載します。

注

エディタが表示されたツールによって、メニューに追加アイテムが含まれる場合があります(たとえばRepeaterでは、URLをリクエストとして貼り付け、現在のアイテムをサイトマップに追加など)。

スキャン / ...に送信

任意のメッセージやメッセージ内の選択した部分を、Burpの他のツールに送信して、さらなる攻撃や分析が実行できます。ツール間のリクエストの送信機能は、Burpのユーザ主導ワークフローの中核をなします。

レスポンスをブラウザで表示

選択したレスポンスをブラウザ内でレンダリングさせ、Burpの内蔵HTMLレンダラーの制限を回避します。(Burpのインスタンスをプロキシに設定した)ブラウザにペーストできる一意なURLが生成され、レスポンスがレンダリングされます。その結果、ブラウザのリクエストがBurpによって処理され(リクエストは元のWebサーバには送信されません)、選択したレスポンスを正確に返し、最初にリクエストされたURLのコンテキストでブラウザがレスポンスを処理します。したがって、レスポンス内の相対リンクは、ブラウザによって正しく処理されます。その結果、ブラウザはレスポンスのレンダリングの課程で追加のリクエスト(画像やCSSなど)を生成する場合があり、これらはBurpの通常の方法で処理されます。

ブラウザでリクエスト

選択したリクエストを(Burpのインスタンスをプロキシに設定した)ブラウザで再送信できます。次のサブオプションがあります:

  • 元のセッションを使う - 元のリクエストにあるCookieヘッダを正確に使ったリクエストを発行します。
  • 現在のブラウザのセッションを使う - ブラウザによって提供されたcookieを使ったリクエストをBurpが発行します。あるユーザコンテキスト(管理者など)で生成されたリクエストを選択し、現在ログインしている別のユーザコンテキスト(一般ユーザなど)でリクエストを再送するなど、アクセス制御のテストを容易にするためにこの機能が使えます。複雑な多段階プロセスを扱っているとき、多段階のプロセスでcookieを何度も繰り返しProxyで修正するより、手動で一連のURLをBurpからブラウザに貼り付ける方が簡単です。

関連ツール

このサブメニューには、関連するタスクを実行するさまざまな便利な機能があります:

  • 参照元検索 - 参照元検索機能で、現在表示されているアイテムにリンクしているHTTPレスポンスを、すべてのBurpツールから検索できます。
  • コンテンツ探索 - コンテンツ探索機能で、巡回もしくはSpiderで表示されているコンテンツからはリンクされていない、コンテンツや機能を探索できます。
  • スケジュールタスク - 指定した時間や間隔で自動的に実行するタスクを作成できます。
  • CSRF PoCジェネレータ - CSRF PoCジェネレータ機能で、ブラウザで表示したときに現在のリクエストが発行されるようなHTMLを作成できます。

リクエストメソッド変更

リクエストに対して、リクエストメソッドのGETとPOSTを自動的に切り替え、関連するすべてのリクエストパラメータがリクエスト内で適切に再配置されます。このオプションを使用すると、入力フィルタのバイパスやクロスサイトスクリプティング攻撃の微調整など、アプリケーションのパラメータ位置の許容範囲を素早くテストできます。

ボディのエンコーディング変更

リクエストに対して、任意のメッセージボディのエンコードを、標準のURLエンコードとマルチパートを切り替えられます。

URLをコピー

現在の完全なURLをクリップボードにコピーします。

curlコマンドをコピー

現在のリクエストを生成するcurlコマンドをクリップボードにコピーします。

ファイルにコピー

ファイルを選択し、現在のメッセージのコンテンツをファイルにコピーできます。これは、クリップボードにコピーすると問題が発生する可能性があるようなバイナリコンテンツに便利です。選択したテキストか、何も選択していない場合はメッセージ全体がコピー処理されます。

ファイルから貼り付け

ファイルを選択し、ファイルの内容をメッセージ内に貼り付けられます。これは、クリップボードから貼り付けると問題が発生する可能性があるようなバイナリコンテンツに便利です。選択したテキストを置き換えるか、何も選択されていなければカーソル位置に挿入されます。

アイテムを保存

選択したリクエストとレスポンスを、レスポンス長、HTTPステータスコード、MIMEタイプなどすべての関連メタデータを含むXMLフォーマットで、選択したファイルに保存します。

選択箇所を変換

これはRawビューにのみ適用されます。このサブメニューのアイテムで、選択したテキストをさまざまな形式で素早くエンコードまたはデコードができます。メッセージが編集可能な場合、テキストが選択されたその場所が変換されます。メッセージが編集可能でない場合は、変換結果がダイアログに表示されます。次のタイプの変換が利用できます:

  • URL - URLエンコードまたはデコードを実行します。主要HTTP特殊文字のみ、すべての文字、すべての文字を2バイトのUnicodeエンコーディングで(たとえばAは%0041に)エンコードするか選択できます。
  • HTML - HTMLエンコードまたはデコードを実行します。主要なHTML特殊文字のみ、すべての文字、すべての文字を10進数の数値文字参照で(たとえばAは&#54;に)、すべての文字を16進数の数値文字参照で(たとえばAは&#x41;に)エンコードするか選択できます。
  • Base64 - Base64エンコードまたはデコードを実行します。
  • 文字列生成 - さまざまな言語で、選択した文字列を動的に構築するコードを生成します。SQLインジェクションのような特定の攻撃で、文字列を動的に構築して入力フィルタを回避する必要がある場合に便利です。使用可能なオプションは、JavaScriptと、Microsoft、Oracle、MySQLプラットフォーム上のSQLです。

入力をURLエンコード

これはRawビューにのみ適用されます。このオプションをオンにすると、&や=などの文字は、入力時に自動的にURLエンコードされた同等の値に置き換えられます。
