その他のユーザオプション

このタブには、ホットキー、プロジェクト自動バックアップ、Proxyインターセプト、REST API、Proxy履歴ログ、一時ファイルの場所、パフォーマンスフィードバックがあります。

ホットキー

一般的なアクションのためのホットキーを設定できます。多くのタイプのアクションにホットキーを割り当てられ、次のカテゴリがあります:

  • Repeaterへ送信など、個々のHTTPリクエストやレスポンスに固有のアクション。
  • Proxyに切り替えなど、グローバルアクション。
  • 切り取り、元に戻すなど、エディタ内のアクション。

多くのホットキーがデフォルトで設定されています。ホットキーをよく使う場合、割り当てられているよりも多くの機能がある点に注意してください。

すべてのホットキーはコントロールキー(またはOSXではコマンドキー)を使用する必要があり、シフトやその他の修飾キーも使用できます。一部のWindowsインストールでは、Ctrl+Altの組み合わせはAltGrと同等として扱われるので、テキストフィールドに入力した文字が現れる可能性がある点に注意してください。

プロジェクト自動バックアップ

プロジェクト自動バックアップは、バックグラウンドで定期的にBurpプロジェクトファイルのコピーを保存します。次のオプションがあります:

  • 自動バックアップを実行するかどうかと、その頻度。
  • スコープ内のアイテムのみ含めるかどうか。
  • バックアップ中に進捗ダイアログを表示するかどうか。
  • Burpの正常終了時にバックアップファイルを削除するかどうか。

REST APIオプション

REST APIを使用して、他のツールとBurp Suiteを統合できます。

注

REST APIは機密性の高い機能とデータを公開します。信頼できないネットワークインタフェースではREST APIサービスを有効にせず、アクセスを許可するクライアントごとに個別のAPIキーを使用するべきです。

次のオプションがあります:

  • サービスが実行されるURL。バインドするポート番号とインタフェースを選択できます。信頼できないネットワークに接続している場合は、ループバックインタフェース以外にバインドしないでください。
  • サービスが現在実行中かどうか。
  • APIキーなしでアクセスを許可するかどうか。このオプションは非推奨です。サービスエンドポイントへのネットワークアクセスができる人は誰でも、Burp内のアクションを起動してしてそのデータにアクセスできることになります。これには、Burpと同じマシンでブラウズする信頼できないWebサイトからのCSRFリクエストも含まれるため、サービスがループバックインタフェースのみでリッスンしている場合でも、常にAPIキーを使用する必要があります。
  • クライアントが使用するAPIキー。異なる用途に別々のAPIキーを作成し、それらの有効または無効を選択できます。APIキーは秘密で、慎重に扱う必要があります。APIキーの値は、作成された時点でしか取得できないことに注意してください。

設定を行うと、APIドキュメントのブラウズができ、[サービスURL]/[APIキー]でAPIとやりとりできます。

Proxyインターセプト

Burp起動時に、Proxyインターセプトを有効にするか設定します。常に有効化、常に無効化、前回終了時の状態を復元、から選択できます。

Proxy履歴ロギング

ターゲットスコープへアイテムを追加したときに、履歴やライブタスクにスコープ外アイテムを送信しないようProxyオプションを自動的に追加するかどうか設定します。この設定により、プロジェクトデータにスコープ外のアイテムが蓄積されるのを避けられます。

一時ファイルの場所

Burpが一時ファイルを保存する場所を設定します。

Burpはデフォルトで、プラットフォームの提供する一時ファイル場所にディレクトリを作成します。これを別のディレクトリを使用するように変更できます - たとえば、別のボリュームや、全ユーザに読取権限がない場所などです。

Mac OS Xでは、システムを休止状態にするとデフォルトの一時ファイル場所がクリアされることがあり、Burpが一時ファイルを失う原因になります。Burpの一時ファイルを格納するカスタムの場所を設定すると、この問題を解決できます。

この設定の変更は、次回のBurp起動時に有効になります。

パフォーマンスフィードバック

Burpのパフォーマンスに関する匿名のフィードバックを送信すると、Burpの向上に貢献できます。

フィードバックにはBurpの内部機能に関する技術情報だけが含まれ、あなたを特定する方法はありません。バグを報告する場合デバッグIDを含めると、あなたのBurpインスタンスで発生した問題の診断の助けになります。

例外をローカルディレクトリに記録する

一部のケースで、匿名フィードバックを有効にしても、パフォーマンスの問題に関するデータを受け取ることができない場合があります。たとえば、厳格なセキュリティポリシーがある場合、サポートチームへのデータ送信がブロックされる場合があります。この場合、例外をローカルディレクトリに記録するオプションを一時的にアクティブにした上で、問題を再現するよう依頼する場合があります。

このオプションがアクティブな間は、指定されたディレクトリのローカルファイルに例外が記録されます。したがって、問題を再現できる場合は、ローカルログファイルのコピーを手動でサポートチームに送信して、サポートチームが問題を特定できるようになります。ファイル名は、現在の日付とデバッグIDを組み合わせたもので、例外が発生したセッションを識別します。Burpの各インスタンスは、個別のログを生成します。

ログエントリには、例外の時刻、簡単な説明、スタックトレースのみが含まれます。スタックトレースは完全に難読化されているため、ファイルから個人データは読み取れません。

注

ログを保存するディレクトリを変更した場合は、問題を再現しようとしているユーザが、指定されたディレクトリへの書き込みアクセス権を持っていることを確認してください。持っていない場合、ファイルは生成されません。

更新設定

デフォルトでは、Burpは利用可能なアップデートを自動的にダウンロードします。新しいアップデートがダウンロードされると、Burpをインストールするために再起動するように求める通知が表示されます。通知を閉じるか、後でを選択すると、次に手動でBurpを再起動したときに更新がインストールされます。

更新をStableまたはEarly Adopterチャネルのどちらからダウンロードするかを選択できます。Burp Suiteの新機能やその他の改善点は、Early Adopterチャネルに最初にリリースされます。Early Adopterバージョンで初期の問題が解決されると、Stableチャネルにリリースされます。デフォルトでは、BurpのすべてのインストールはStableチャネルから更新されます。ただし、インストールごとに使用するチャネルを手動で選択もできます。

注

当社のWebサイトから任意のバージョンのBurp Suiteを手動でダウンロードできます。Early Adopterチャネルでのみ利用可能なバージョンを手動でダウンロードしてインストールすると、更新チャネルが自動的にEarly Adopterに切り替えられることに注意してください。

Early AdopterチャネルからStableチャネルに移動したり、以前のStableバージョンをダウンロードするなど、古いバージョンのBurp Suiteに戻した場合、プロジェクトファイルが必ずしも以前のバージョンで機能するとは限りません。Burpは、プロジェクトファイルが現在のインストールと互換性がない可能性があることを警告します。これを行うと、プロジェクトファイルが損傷する可能性があります。古いバージョンのBurp Suiteに移行する場合は、新しいプロジェクトファイルの使用のみを推奨します。

必要に応じて、自動更新を有効にするチェックボックスをオフにすると、自動更新を無効にできます。この場合、アップデートが利用可能になると通知されますが、ダウンロードやインストールはしないことを選択できます。

他のユーザオプションとは異なり、更新設定はBurpの現在のインストールにのみ適用されることに注意してください。複数のインストールを使用する場合は、それぞれの設定を変更する必要があります。これにより、ユーザはBurp SuiteのStableバージョンとEarly Adopterバージョンの両方を同時にインストールできます。

注

JARファイルからBurpを起動した場合、自動更新機能は利用できません。

メッセージ検索

メッセージエディタウィンドウの下部にある検索バーのデフォルトを制御します。各オプションは、検索バー自体の歯車アイコンからオンとオフを切り替えられますが、ここでのオプションは、次の設定で使用されるデフォルトを制御します。

  • 大文字小文字を区別 - デフォルトで、大文字と小文字を区別するかどうかを指定します。
  • 正規表現 - デフォルトで、検索語が正規表現かリテラル文字列かを指定します。
  • テキスト変更時にマッチした箇所に自動スクロール - デフォルトで、新しいテキストが表示されたときに、最初に強調表示された一致箇所までテキストエディタが自動でスクロールするかどうかを指定します。これはたとえば、Proxy履歴でレスポンスから特定の文字列を探してアイテムを順に見ている際に便利です。このオプションを選択して新しいアイテムを選択すると、最初の検索一致箇所に自動で表示がスクロールします。

内蔵ブラウザ

Burpの内蔵Chromiumブラウザでの設定と履歴が、永続的であるかどうかを決定します。これはデフォルトで有効になっています。つまり、Chromiumを好きなように設定でき、お気に入りの拡張機能もインストールできます。これらの変更は、内蔵ブラウザを開くたびに復元されます。

このオプションを無効にすると、内蔵ブラウザは常に完全にクリーンな状態で開きます。Chromiumを閉じると、設定に加えた変更はすべて失われ、ブラウザを起動するたびに拡張機能を再インストールする必要があります。

学習タブ

学習タブをメインバーに表示するかどうか制御します。ここには、便利なチュートリアルへのリンクやその他のリソースへのリンクがあり、Burp Suiteを使用開始し、新機能を発見し、スキルを次のレベルに引き上げるのに役立ちます。

タスク

既存のプロジェクトを最初に開いたときのスタートアップウィザードでの、自動タスクを一時停止するオプションのデフォルト状態を制御します。

このチェックボックスをオンにすると、既存の自動化されたタスクが起動時に一時停止します。チェックボックスをオフにすると、既存のタスクが自動的に再開されます。
