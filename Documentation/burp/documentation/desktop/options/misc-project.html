Burp Collaboratorサーバ

Burp Collaboratorは、Burpが多くの脆弱性を発見するのに役立つ外部サービスです。Burp Collaboratorの機能と利用方法に関する詳細は、Burp Collaboratorドキュメントを参照してください。

注: Burp Collaboratorの機能は、慎重に検討しないと問題が起こる場合があります。Burp Collaboratorの機能と代替手段を完全に理解し、利用による自分自身や組織への影響を考慮しておく必要があります。

Burp Collaboratorを使用するために、次のオプションがあります:

  • デフォルトのCollaboratorサーバを使用する - このデフォルトオプションは、PortSwiggerが提供する公開Collaboratorサーバを使用します。このサーバは、これを使用する全てのBurpユーザで共用です。もし公開Collaboratorサーバの停止や不具合があると、BurpのCollaboratorに関連する機能の有効性が失われる可能性があります。そのため、PortSwiggerはこのサーバのパフォーマンスまたは可用性について保証しません。
  • Burp Collaboratorを使用しない - Collaboratorに関連した機能はどれも有効になりません。
  • プライベートCollaboratorサーバを使用する - 独自のCollaboratorサーバのインスタンスを使用します。これを行うには、プライベートCollaboratorサーバのデプロイドキュメントを参照してください。

プライベートCollaboratorサーバを使用する場合、その位置について詳細を設定する必要があります。次のオプションがあります:

  • サーバの場所 - サーバのドメイン名またはIPアドレスです。サーバをIPアドレスで指定する場合、DNSに依存するBurpのCollaborator関連機能は利用できません。詳細は、Burp Collaboratorドキュメントを参照してください。
  • ポーリング場所(オプション) - このオプションフィールドには、プライベートCollaboratorサーバがポーリングリクエストに応答する位置を指定します。必要であればCollaboratorサーバが、通信の受信とポーリングリクエストの応答を、異なるネットワークインタフェースで行うよう設定できます。ポーリング位置をホスト名かIPアドレスで、オプションでコロンで区切ったポート番号を指定できます。例えば、10.20.30.40:8008 です。

さらに次のオプションがあります。

  • 暗号化されていないHTTPでポーリングする - デフォルトでBurpは、HTTPS上でCollaboratorサーバをポーリングし、中間者攻撃を防ぐためSSLを強制します。ネットワークやその他の制限でBurpがHTTPS上で直接ポーリングできない場合、暗号化されていないHTTP上でのポーリングもできます。
  • ヘルスチェック実行 - このボタンをクリックすると、設定したCollaboratorサーバのヘルスチェックを実行するダイアログが開きます。様々なネットワークサービスが通信できるかどうか、通信の詳細をポーリングで取得できるかどうかを確認します。これらのテストから、BurpがCollaboratorの機能を全てまたは一部利用できるか、または全く利用できないか判断できます。

ロギング

HTTPリクエストとレスポンスのロギングを制御します。ロギングは、ツールごと、または全Burpトラフィックに対して設定できます。
