オプション: 表示

このタブには、ユーザインタフェース、HTTPメッセージ表示、文字セットハンドリング、HTMLレンダリングの設定があります。

ユーザインタフェース

Burpユーザインタフェースの外観を制御します。明るいテーマか暗いテーマかを選択し、UI全体で使用するフォントサイズを調整できます。HTTPメッセージの表示に使用するフォントは別途制御されていることに注意してください。

Burp Suiteでダークモードを有効にする方法

Burp Suiteのダークテーマを有効にするには、ユーザ設定 > ディスプレイタブに移動します。ユーザインタフェースで、Darkテーマを選択します。

HTTPメッセージ表示

HTTPメッセージエディタにHTTPメッセージを表示する方法を制御します。フォントのフェイスとポイントサイズ、フォントスムージングを使用するかどうか設定できます。システム上にあるすべての等幅フォントのリストから選択できます。

次の追加オプションがあります:

  • リクエストの構文を色付けする - JavaScript、JSON、CSSなどを含むリクエストついてシンタックスハイライトを行うかどうか制御します。
  • レスポンスの構文を色付けする - JavaScript、JSON、CSSなどを含むレスポンスついてシンタックスハイライトを行うかどうか制御します。
  • デフォルトで整形表示 - Burpが自動的に、整形されていないデータ、マークアップ、コードに対して、正規化された改行やインデントを追加して整形して表示するかどうかを制御します。現在、JSON、XML、HTML、CSS、JavaScriptで整形表示がサポートされています。

文字セット

HTTPメッセージを生で表示する際に、Burpがさまざまな文字セットを処理する方法を制御します。使用可能なオプションは次の通りです:

  • メッセージヘッダに基づいて各メッセージの文字セットを自動的に認識する。これがデフォルトオプションで、さまざまな文字セットを使用するメッセージを同時に作業できます。
  • すべてのメッセージでシステムのデフォルト文字セットを使用する。
  • メッセージを(ASCIIエンコードを使用して)生バイト列で表示し、拡張文字セットの処理を行わない。
  • すべてのメッセージに指定した文字セットを使用する。

HTTPヘッダは常に生形式で表示されます - 文字セットエンコーディングの設定はメッセージボディにのみ適用されます。

一部の文字セットに必要な字形がすべてのフォントでサポートされていないことに注意してください。拡張または特殊な文字セットを使用する場合は、まずCourier NewやDialogなどのシステムフォントを試してみてください。

HTMLレンダリング

HTTPメッセージエディタ内のRenderビューは、ブラウザに表示されるのとほぼ同じようにHTMLコンテンツを表示します。HTMLコンテンツを完全にレンダリングするために必要な追加のHTTPリクエスト(たとえば、埋め込み画像など)を送信するか制御します。このオプションはHTMLレンダリングの品質と速度のトレードオフになり、対象アプリケーションへのさらなるリクエスト送信を避けたい場合に使用してください。

Inspectorの表示設定

Inspectorには表示設定メニューがあり、画面上のInspectorパネルの表示位置や、どのウィジェットを表示するかを設定できます。

Inspectorの表示設定の詳細は、Inspectorの表示設定セクションを参照してください。
